# S1: 需求分析 — 移动端抽屉优化与图标生成

## 需求来源
- S0 文档: `S0-原始需求确认.md`
- 已确认需求摘要: ①移动端底部抽屉增加高度+降低留白 ②生成 PWA 图标 ③合并到主干

## 问题描述

### 需求1：移动端底部抽屉优化
当前移动端底部抽屉（GisMobileNav.vue）高度仅 35%，且容器内存在多处留白，导致有效内容展示区域不足。

### 需求2：PWA 图标生成
005 任务已更新 pwa-icon.svg（线宽/半径/不透明度 +30%），但 pwa-192x192.png 和 pwa-512x512.png 仍是旧 SVG 生成的，需用 gen-pwa-icons.mjs 重新生成。

### 需求3：合并到主干
feature/share-target 分支含 004+005 两个任务成果，需合并到 main。

## 根因分析

### 需求1 根因
移动端底部抽屉留白来自多个层级：

1. **el-drawer 自身默认 padding**：Element Plus 的 el-drawer__body 默认有 padding（通常 20px），App.vue 的移动端优化只覆盖了 el-dialog，未覆盖 el-drawer
2. **drawer-handle 留白**：`GisMobileNav.vue:172` margin: 8px auto 4px，上下共 12px
3. **drawer-body padding**：`GisMobileNav.vue:180` padding: 2px（虽小但叠加其他留白）
4. **抽屉高度不足**：`GisMobileNav.vue:89` size="35%"，用户要求 50%

### 需求2 根因
PNG 图标与 SVG 源文件不同步，005 修改了 SVG 但未重新生成 PNG。

### 需求3 根因
功能分支开发完成，需合并到主干发布。

## 现状分析（已有基础设施 vs 缺失部分）

### 已有
- `GisMobileNav.vue` 抽屉组件完整，direction="btt" 底部上滑
- `App.vue:117-152` 已有移动端 el-dialog 留白优化模式，可参考扩展到 el-drawer
- `scripts/gen-pwa-icons.mjs` 图标生成脚本完整，使用 sharp 库
- `public/pwa-icon.svg` 已在 005 更新

### 缺失
- el-drawer 在移动端的留白优化（App.vue 未覆盖）
- 抽屉高度未调整
- PNG 图标未重新生成

## 影响范围

### 需求1 影响
- 文件: `src/components/data/GisMobileNav.vue`（抽屉高度 + drawer-handle/body 留白）、`src/App.vue`（全局 el-drawer 移动端留白优化）
- 模块: 移动端导航抽屉
- 场景: 移动端用户使用底部抽屉查看坐标系/要素/校验/导出
- 桌面端: 无影响（el-drawer 移动端优化在 @media max-width:767px 内）

### 需求2 影响
- 文件: `public/pwa-192x192.png`、`public/pwa-512x512.png`（二进制文件更新）
- 模块: PWA 图标
- 场景: PWA 安装时显示的图标

### 需求3 影响
- 分支: feature/share-target → main
- 场景: 主干发布

## 约束条件
- 不破坏桌面端布局
- el-drawer 移动端优化仅在 @media max-width:767px 内生效
- 不修改抽屉内子组件
- gen-pwa-icons.mjs 依赖 sharp 库，需确认已安装

## 验证结论
- sharp 库是否安装：待 S5 执行时验证（package.json devDependencies）

## 最小改动假设（微创基线）

### 最少需改动的文件/函数
1. `src/components/data/GisMobileNav.vue` — size 35%→50%、drawer-handle margin 缩减、drawer-body padding 缩减
2. `src/App.vue` — @media 移动端区块内追加 el-drawer 留白优化（el-drawer__body padding）
3. `public/pwa-192x192.png` + `public/pwa-512x512.png` — 运行脚本生成（二进制更新，非代码改动）

### 明确不需要改动的区域
- 抽屉内子组件（GisDataTransformer/GisFeatureTree/GisDataValidator/GisDataExport）
- gen-pwa-icons.mjs 脚本本身
- pwa-icon.svg（005 已改）
- mobile-tab-bar（底部导航栏，非抽屉部分）

### 可复用能力
- 复用 App.vue 已有的 @media (max-width:767px) 移动端优化模式
- 复用 gen-pwa-icons.mjs 脚本

### 选择该最小范围的理由
三个需求各自有明确的单点操作位置，需求1 仅改样式数值和追加全局样式，需求2 仅运行脚本，需求3 仅 git 操作。

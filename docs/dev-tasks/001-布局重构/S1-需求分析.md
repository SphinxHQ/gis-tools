# S1: 需求分析 — 布局重构

## 需求来源

- S0 文档: `S0-原始需求确认.md`
- 已确认需求摘要: 将四层 Tabs 嵌套重构为「顶栏 + 可折叠左面板 + 主内容区」扁平化布局，深度适配移动端，4 档断点响应式

## 问题描述

当前 GIS 数据工具存在三大体验问题：

1. **四层 Tabs 嵌套**：`GisDataSetTabs → GisDataTransformer → GisDataViewer → GisDataInspactor`，用户需要点击 4 次 Tab 才能到达要素编辑/校验功能，移动端灾难
2. **固定侧边栏侵占横向空间**：
   - MapDrawer 左侧 200px
   - GisDataInspactor Splitpanes 22%
   - GisDataViewer FeatureSplit 220px
   - 手机端（375px）这些固定宽度直接占满屏幕
3. **导出功能分散**：GeoJson/Wkt/电子报盘 分散在 GisDataViewer 的不同 Tab，用户需要来回切换

## 根因分析

| 问题 | 根因 |
|------|------|
| 四层 Tabs 嵌套 | 早期按「数据集→坐标系→查看→检查」线性流程设计，每层都用 el-tabs 承载，未考虑层级扁平化 |
| 固定侧边栏 | 桌面优先设计，未做响应式断点，所有尺寸都用固定 px |
| 导出分散 | GisDataViewer 把「查看」和「导出」混在一起，职责不清 |

## 约束条件

1. **兼容性**：不破坏现有数据流（useGisDataStore），不修改底层解析逻辑
2. **性能**：地图组件不重复挂载，避免移动端卡顿
3. **Element Plus 规范**：使用 el-drawer/el-segmented/el-tabs 等原生组件，不引入新依赖
4. **向后兼容**：保留所有现有功能（导入/转换/查看/校验/导出/绘制）
5. **主题**：适配深色/浅色主题（themeMode）
6. **断点**：4 档断点 ≥1280/1024-1279/768-1023/<768

## 验证结论

- 已验证：useGisDataStore 数据流清晰，UI 重构不影响数据层
- 已验证：Element Plus 提供 el-drawer（抽屉）、el-segmented（分段器）、el-tabs（标签页）、el-affix（固定）等响应式组件
- 已验证：Vue 3 composable 可封装断点逻辑（window.matchMedia）
- 已验证：现有 GisMapTianditu 可通过 props 控制 projection，支持复用

## 最小改动假设（微创基线）

### 最少需改动的文件/函数

1. **新建** `src/composables/useBreakpoint.ts`（~50 行）
   - 4 档断点 composable，基于 window.matchMedia

2. **重构** `src/components/data/GisData.vue`（主布局容器）
   - 从「toolbar + GisDataSetTabs」改为「顶栏 + 左面板 + 主内容区」
   - 顶栏：el-select 数据集切换 + 导入按钮 + 主题切换
   - 左面板：GisDataPanel（5 Tab）
   - 主内容区：地图 + 数据信息概览
   - 移动端：底部 GisMobileNav + 上滑面板

3. **新建** `src/components/data/GisDataPanel.vue`（左面板 5 Tab 容器，~100 行）
   - el-tabs 竖向 5 Tab：数据源/坐标系/要素/校验/导出
   - 每个 Tab 承载对应组件

4. **新建** `src/components/data/GisMobileNav.vue`（移动端底部导航，~80 行）
   - 底部 5 Tab + 上滑面板（el-drawer direction="btt"）

5. **重构** `src/components/data/GisDataTransformer.vue`
   - 去 tabs 化：editableTabs 改为 el-select 下拉 + 当前 CRS 信息卡片
   - 转换链改为面包屑式展示

6. **拆分** `src/components/data/GisDataViewer.vue`
   - 数据信息 → 主内容区概览组件
   - GeoJson/Wkt/电子报盘 → 迁入「导出」Tab

7. **拆分** `src/components/data/GisDataInspactor.vue`
   - 结构树 → 「要素」Tab
   - 校验 → 「校验」Tab
   - 地图 → 主内容区

8. **重构** `src/components/data/MapDrawer.vue`
   - 200px 固定侧栏 → el-drawer 抽屉化

### 明确不需要改动的区域

- `src/composables/gisDataStore.ts`（数据流不变）
- `src/composables/eventBus.ts`（事件总线不变）
- `src/components/gismap/GisMapTianditu.vue`（地图核心不变）
- `src/components/data/GisFeatureEditor.vue`（要素编辑器不变）
- `src/components/data/GisCrsSelector.vue` / `GisCrsTransformSelector.vue`（CRS 选择器不变）
- `src/components/data/DataFormat.ts` 等解析逻辑（不变）
- `src/components/editor/monacoSetup.ts`（Monaco 配置不变）
- `src/composables/dark.ts`（主题切换不变）
- `index.html` / `public/entry-loader-worker.js`（loading 不变）

### 可复用能力

- `useGisDataStore`：数据集管理（直接复用）
- `GisDataReader`：数据读取（内容迁入左面板，组件本身可复用或拆分）
- `GisCrsSelector` / `GisCrsTransformSelector`：CRS 选择（直接复用）
- `GisFeatureEditor`：要素编辑（直接复用）
- `GisMapTianditu`：地图（直接复用，主内容区挂载）
- `eventBus`：地图事件通信（直接复用）
- Element Plus：el-drawer / el-segmented / el-tabs / el-select / el-affix（原生组件）

### 选择该最小范围的理由

1. **数据流不变**：useGisDataStore 已经是 composable 风格，UI 重构不需要动数据层
2. **组件复用**：GisDataReader/GisCrsSelector/GisFeatureEditor/GisMapTianditu 都可以原样复用，只需调整挂载位置
3. **拆分而非重写**：GisDataViewer/GisDataInspactor 拆分时保留核心逻辑（transformGeometry/inspectFeature/handleValid 等），只调整模板结构
4. **新增最少**：仅新建 useBreakpoint + GisDataPanel + GisMobileNav 三个文件，其余都是现有文件重构

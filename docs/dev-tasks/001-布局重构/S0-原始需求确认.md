# S0: 原始需求确认 — 布局重构

## 用户原始需求

> 我感觉我的三层tabs嵌套很不好，特别手机用户太灾难了。绘制图形的布局也是。左侧固定侧边手机下面占满了都块。
>
> 既然使用 Element Plus 且需要深度兼顾手机端操作，我们的重构方向就可以利用 Element Plus 自带的响应式组件和断点工具（如 el-drawer、el-segmented 和响应式隐藏类）。
>
> **桌面端布局**：
> - 顶栏：数据集切换（从 tabs 改为 el-select 下拉）+ 导入按钮 + 主题切换
> - 左侧可折叠面板（展开 320px / 折叠 48px）：5 个 Tab（数据源/坐标系/要素/校验/导出）
> - 主内容区：地图视图 + 数据信息概览
>
> **移动端布局**（<768px）：
> - 地图优先（map-first）
> - 底部 Tab 导航
> - 上滑面板（slide-up panel）展示详情
> - 抽屉化（drawer-ization）侧边栏
>
> **断点设计**（4 档）：
> - ≥1280px：桌面完整布局（左面板 320px 展开）
> - 1024-1279px：桌面紧凑布局（左面板 280px）
> - 768-1023px：平板布局（左面板抽屉化，主内容全宽）
> - <768px：移动端布局（底部 Tab + 上滑面板）
>
> **导出转换合并**：将 GisDataViewer 中的 GeoJson/Wkt/电子报盘 合并到「导出」Tab
>
> 按照 Element Plus 的常见手机端/桌面端的设计规范进行排版和定制规则设计，先设计再执行。临时文档放进 docs。
>
> 一次性无人工介入走完所有流程。可以多在互联网上进行搜索当前的最新设计思路和规范进行辅助。

## 代码初步定位

### 当前四层 Tabs 嵌套结构

```
GisData.vue (主页面容器)
└── GisDataSetTabs.vue (第1层: 数据集切换 el-tabs type="card" editable)
    └── GisDataTransformer.vue (第2层: 坐标系转换 el-tabs type="card")
        └── GisDataViewer.vue (第3层: 数据信息/编辑&查看/GeoJson/Wkt/电子报盘 el-tabs)
            └── GisDataInspactor.vue (第4层: 结构/校验/要素列表 el-tabs type="border-card" + Splitpanes)
```

### 固定侧边栏问题位置

1. **MapDrawer.vue** `src/components/data/MapDrawer.vue:128-140`
   - `.md-left { width: 200px }` + `.md-right { width: calc(100% - 200px) }`
   - 手机端 200px 左侧栏占据过多横向空间

   - `<Splitpanes>` 22%/78% 分割
   - 左侧 22% 在手机端仍然占用横向空间

3. **GisDataViewer.vue FeatureSplit** `src/components/data/GisDataViewer.vue:491-523`
   - `.feature-split-sidebar { width: 220px; min-width: 180px }`
   - 要素卡片列表固定 220px 宽度

### 数据流

- `useGisDataStore()` (`src/composables/gisDataStore.ts`): 管理 datasets/activeId
- props 传递: `GisDataSetTabs → GisDataTransformer(:data) → GisDataViewer(:data, :transform-chain) → GisDataInspactor(:data)`
- 事件冒泡: `transform-crs` / `reset-crs` / `navigate-chain` 从 GisDataViewer 冒泡到 GisDataTransformer

## AI 复述后的需求

将当前 GIS 数据工具的四层 Tabs 嵌套结构重构为「顶栏 + 可折叠左面板 + 主内容区」的扁平化布局，并深度适配移动端。

**核心目标**：
1. **降维拍平**：消除四层 Tabs 嵌套，改为左侧面板 5 个 Tab（数据源/坐标系/要素/校验/导出）+ 主内容区
2. **横向妥协**：固定侧边栏在移动端抽屉化，释放横向空间
3. **导出合并**：将 GisDataViewer 的 GeoJson/Wkt/电子报盘 合并到「导出」Tab
4. **响应式断点**：4 档断点（≥1280/1024-1279/768-1023/<768）
5. **移动端优先**：<768px 采用 map-first + 底部 Tab + 上滑面板

## 需求边界

### 本次包含

1. 创建 `useBreakpoint` composable（4 档断点响应式工具）
2. 重构 `GisData.vue` 为顶栏 + 可折叠左面板 + 主内容区布局
3. 顶栏：数据集切换从 tabs 改为 el-select 下拉 + 导入按钮
4. 左面板 5 Tab 框架：数据源/坐标系/要素/校验/导出
5. 左面板 Tab1「数据源」：迁入 GisDataReader 内容（上传/文本/绘制/历史）
6. 左面板 Tab2「坐标系」：GisDataTransformer 去 tabs 化，改为面板内操作
7. 左面板 Tab3「要素」+ Tab4「校验」：拆分 GisDataInspactor
8. 左面板 Tab5「导出」：合并 GisDataViewer 的 GeoJson/Wkt/电子报盘
9. 主内容区：地图视图 + 数据信息概览
10. 移动端适配：底部 Tab 导航 + 上滑面板 + 抽屉化

### 本次不包含

1. 不修改 `useGisDataStore` 数据流（UI 重构，数据流保持不变）
2. 不修改底层解析逻辑（SimpleDataFormat/ShapefileParser 等）
3. 不修改地图核心组件（GisMapTianditu）
4. 不修改 Monaco 编辑器配置
5. 不修改 loading 动画和进度算法
6. 不新增第三方依赖（仅使用 Element Plus 现有组件 + Vue 原生能力）

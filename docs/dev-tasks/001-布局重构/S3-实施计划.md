## 修改项清单

### M1: 新建 useBreakpoint composable

- 文件: `src/composables/useBreakpoint.ts`（新建）
- 修改内容:
  - 创建 4 档断点响应式 composable
  - 断点定义：`xl`（≥1280px）/ `lg`（1024-1279px）/ `md`（768-1023px）/ `sm`（<768px）
  - 导出：`current`（当前断点）、`isMobile`（<768px）、`isTablet`（768-1023px）、`isDesktop`（≥1024px）、`panelWidth`（左面板宽度：xl=320/lg=280/md=0/sm=0）
  - 基于 `window.matchMedia` + `resize` 监听
- 影响面: 新增文件，无现有代码影响
- 验证方法: 在浏览器控制台调用 `useBreakpoint()` 验证断点切换
- 回滚方式: 删除文件

## 风险与回滚总述

### 风险

1. **地图实例迁移**（M7）：GisDataInspactor 的地图迁移到主内容区，需确保 eventBus instanceId 正确
   - 缓解：主内容区地图使用固定 instanceId，所有事件通信保持一致
2. **FeatureSplit 响应式**（M6）：220px 侧栏在移动端需抽屉化
   - 缓解：移动端隐藏侧栏，改为下拉选择要素
3. **数据流保持**（M2-M9）：UI 重构不能破坏 useGisDataStore 数据流
   - 缓解：所有组件仍通过 props 接收 data，不修改 store

### 回滚

- 每个修改项独立可回滚（git checkout 单文件）
- 整体回滚：git checkout 所有修改文件 + 删除新建文件

# S3: 实施计划 — 分享上传修复

## 修改项清单

### M1: el-upload 添加 accept 属性
- 文件: `src/components/data/GisDataReader.vue:226-231`
- 修改内容: 在 `el-upload` 组件上添加 `:accept` 属性，值为解析器支持的所有空间数据文件后缀和 MIME 类型
- accept 值（覆盖 GisDataType 全部格式）:
  ```
  .geojson,.json,.wkt,.txt,.csv,.shp,.zip,.dxf,.exf,.wkb,
  application/json,application/zip,application/octet-stream,
  text/plain,text/csv,application/dxf
  ```
- 影响面: 仅影响文件选择器的类型过滤，不改变 handleFileChanged 处理逻辑。桌面端无影响（accept 仅作过滤提示），移动端将显示"文件"分类
- 验证方法: ①桌面端构建后点击上传，确认仍可选择所有文件 ②移动端（待用户验证）点击上传应出现"文件"分类，可浏览选择 .geojson/.shp 等
- 回滚方式: 删除 accept 属性

### M2: manifest 显式补充 start_url 和 scope
- 文件: `vite.config.ts:163`（manifest 对象内，display 字段后）
- 修改内容: 添加 `start_url: '/gis-tools/'` 和 `scope: '/gis-tools/'`
- 影响面: 与 VitePWA 自动补充的值一致，不改变构建产物行为，仅让源码配置自文档化
- 验证方法: 构建后检查 dist/manifest.webmanifest 中 start_url 和 scope 值不变
- 回滚方式: 删除两行配置

### M3: pwa-icon.svg 视觉参数调整
- 文件: `public/pwa-icon.svg`
- 修改内容:
  - `stroke-width="16"` → `stroke-width="21"`（16 × 1.3 = 20.8，取 21）
  - 5 个 `<circle>` 的 `r="20"` → `r="26"`（20 × 1.3 = 26）
  - `fill="rgba(234,88,12,0.15)"` → `fill="rgba(234,88,12,0.45)"`（0.15 + 0.30 = 0.45）
- 影响面: 仅 SVG 源文件视觉参数。注意：pwa-192x192.png / pwa-512x512.png 是独立 PNG 文件，本次不改 PNG（需用户后续用 SVG 重新生成 PNG，或接受 PNG 暂不更新）
- 验证方法: 读取修改后的 SVG 确认数值正确，浏览器打开 SVG 视觉确认
- 回滚方式: 恢复原数值

## 代码初步定位

### 问题1：上传文件 accept 缺失
- 相关文件: `src/components/data/GisDataReader.vue:226-247`
- 初步观察: `el-upload` 组件未设置 `accept` 属性。移动端浏览器在文件选择器无 accept 约束时，默认只显示图片/视频/音频等媒体类型，导致用户无法选择 .geojson/.shp/.dxf 等空间数据文件
- 解析器支持的格式（来自 `src/components/data/GisDataInfo.ts` GisDataType 枚举 + `DataFormat.ts` getDataType）：
  - GeoJson: .geojson .json
  - WKT: .wkt .txt
  - WKB: .wkb
  - Exchange/电子报盘: .txt（[属性描述] 开头）
  - CSV: .csv
  - ShapeFile: .shp
  - ShapeZip: .zip
  - EXF: .exf
  - DXF: .dxf

### 问题2：分享面板无 PWA
- 相关文件: `vite.config.ts:155-228`（VitePWA manifest 配置）
- 初步观察: manifest 配置中**未显式设置 `start_url` 和 `scope`**。VitePWA 构建时会自动补充（构建产物 manifest.webmanifest 中已有 `"start_url":"/gis-tools/","scope":"/gis-tools/"`），但需 S1 进一步确认是否是根因。另一可能原因：已安装的 PWA 读取的是旧 manifest（004 之前无 share_target），需用户重装/更新 PWA 后才会注册分享目标

### 需求3：PWA 图标调整
- 相关文件: `public/pwa-icon.svg`
- 当前数值:
  - 线宽 stroke-width: 16
  - 节点半径 r: 20
  - 中间填充不透明度: rgba(234,88,12,0.15)
- 目标数值（+30%）:
  - 线宽: 16 × 1.3 = 20.8 → 取 21
  - 节点半径: 20 × 1.3 = 26
  - 中间填充不透明度: 0.15 + 0.30 = 0.45（用户表述"+30%"理解为绝对值加 0.30）

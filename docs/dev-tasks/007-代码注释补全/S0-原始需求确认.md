## MUST NOT 级别约束
- **[MUST NOT] 不允许使用任何脚本批量处理**：所有注释必须逐文件手动编写，由 AI 逐个读取文件内容、理解代码逻辑后编写注释。禁止使用 PowerShell/Shell 脚本、sed/awk、正则替换等批量注入方式。原因：脚本批量处理无法理解代码语义，会导致注释质量低下、type/方法注释缺失。
- **[MUST NOT] 注释掉无效代码时禁止使用 @deprecated 等说明性注释**：必须使用块注释 `/* ... */` 将无效代码完全包裹，确保语法树干净。不允许仅添加 @deprecated 标记而保留代码在语法树中。
- **[MUST NOT] 无效代码不得残留在语法树中**：注释掉的代码必须从 TypeScript/Vue 编译视角完全消失，不得产生任何类型、导出或副作用。

## 清理无效代码规则
- 注释方式：全部使用块注释 `/* ... */`，不使用行注释 `//`
- 分散代码块：如果文件中无效代码分散，尝试合理集中后一同块注释
- 完整性：无效代码必须全部注释掉，包括调用链和引用链
- 保留文件：即使文件内容全部被注释掉，也不删除文件本身

## 五、二期方案重大调整：移除 FAB（2026-06-25）

### 5.1 调整背景
用户反馈："已经进入数据查看阶段你还弄个fab让我绘制？完全是为了fab来左的。完全没用。"

经分析，FAB 在数据查看场景属于**场景错配**：
- 数据查看页面（GisData.vue）核心任务是浏览检视，非绘制
- 绘制功能在 MapDrawer.vue（MapControlPanel）中已有完整实现
- FAB 的 Material Design 风格与 Element Plus 整体风格冲突

### 5.2 调整内容
**移除数据查看页面的 FAB 组件**：
| 文件 | 改动 |
|---|---|
| `src/components/data/GisData.vue` | 移除 GisMapFab import、FabItem type import、fabItems 数组、handleFabSelect 方法、mobile-fab-area CSS 及模板引用 |

保留 `src/components/gismap/GisMapFab.vue` 组件文件备用，但不再在数据查看页面引入。

### 5.3 移动端布局溢出修复
移除 FAB 过程中发现并修复了 3 个布局问题（详见 S6 §6）：
1. 底部 Tab 栏宽度只有 99px → 修复为全屏宽度
2. 内容区高度为 0（地图不可见）→ 修复 navArea 固定高度
3. Tab 栏高度溢出 1px → 添加 box-sizing: border-box

### 5.4 最终验收结论

| 验收项 | 状态 | 说明 |
|---|---|---|
| 单一代码路径原则 | ✅ | unified-layout 弹性布局，CSS 控制响应式 |

| 移动端底部 Tab 栏 | ✅ | 全宽显示，56px 高度，4 Tab 均分 |

| TypeScript 编译 | ✅ | vue-tsc 0 错误 |

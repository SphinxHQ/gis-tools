# S1: 需求分析 — BottomSheet 与 FAB 改造

## 需求来源
- S0 文档: `S0-原始需求确认.md`
- 已确认需求摘要: 按 `ux-renovation-plan.md` 二期方案，将移动端 `el-drawer` 抽屉改造为符合 `geo-compacter.md` 规范的 Bottom Sheet 组件，新增 FAB 悬浮按钮承载地图控件，Tab 内容改用 `v-show` 保留状态，提取共享 Tab 逻辑为 `useDataPanelTabs` composable。参考 `phase1-inspection-rules-v2.md` 的 V4 检验经验制定二期检验规则。

## 问题描述

| # | 违规项 | 位置 | 违反理论 | 严重度 |
|---|--------|------|---------|--------|
| 2.1 | 抽屉重挂载：`v-if` 切 Tab 导致四个重型组件重新挂载 | `GisMobileNav.vue` L102-130 | Norman 反馈原则、Hick's Law | 🔴 |
| 2.2 | 地图控件不可达：固定左上角红色困难区，无触控热区 | `MapControlPanel.vue` L96-99 | Hoober 拇指热区、Fitts's Law | 🔴 |

| 2.4 | Tab 字号过小：10px 低于 iOS Safari 11px 阈值 | `GisMobileNav.vue` L222 | 字体排版规范 | 🟡 |
| 2.5 | GisDataPanel 与 GisMobileNav 逻辑重复 | 两文件 | DRY 原则 | 🟢 |

## 根因分析

### 根因 1: el-drawer 不符合 Bottom Sheet 规范
- **症状**: 使用 `el-drawer direction="btt" size="50%"` 模拟底部抽屉
- **根因**: el-drawer 是通用抽屉组件，设计目标是侧边栏/遮罩弹窗，不具备 Bottom Sheet 的核心能力：
  - 无吸附点（snap points）—— 无法在 25vh/40vh/80vh 自动停留
  - 无拖拽手柄交互 —— `.drawer-handle` 仅是视觉装饰（36×4px），不可拖拽
  - 无半屏/全屏切换 —— size 固定 50%，无法动态调整
  - 无 Standard easing 动画 —— el-drawer 使用自有过渡曲线
- **影响**: 用户无法按需调整面板高度，不符合移动端"Overview first → details-on-demand"交互范式

### 根因 2: v-if 导致组件重挂载
- **症状**: Tab 内容用 `v-if="activeTab === 'crs'"` 等条件渲染
- **根因**: v-if 会销毁/重建 DOM 与组件实例，导致：
  - GisDataTransformer（含 CRS 转换链状态）状态丢失
  - GisFeatureTree（含要素树展开状态）状态丢失
  - GisDataValidator（含校验结果）状态丢失
  - GisDataExport（含导出配置）状态丢失
- **影响**: 每次切 Tab 等于重新进入，违反 Norman 的"概念模型一致性"

### 根因 3: 地图控件位于拇指困难区
- **症状**: `MapControlPanel.vue` 通过 `padding: 10px 0 0 40px` 固定左上角
- **根因**: 未按 Hoober 拇指热区理论布局，左上角属于红色困难区（单手操作需调整握持）
- **加剧因素**: 按钮为原生 `<button>` 无 `min-height/min-width` 设定，热区不足 48dp
- **影响**: 移动端单手操作时地图控件难以触达

### 根因 4: Tab 字号低于阈值
- **症状**: `.tab-label { font-size: 10px }`
- **根因**: 未遵循"移动端最小字号 ≥ 11px"规范
- **影响**: iOS Safari 会自动放大 < 11px 文字，导致布局偏移

### 根因 5: Tab 逻辑重复
- **症状**: GisDataPanel 与 GisMobileNav 各自维护 activeData/transformChain/hasData/hasActiveData
- **根因**: 未提取共享 composable，违反 DRY 原则
- **影响**: 状态逻辑修改需同步两处，维护成本高

## 约束条件

### 设计规范约束（来自 geo-compacter.md）
1. **单一代码路径**：禁止 `if (isMobile)` 分支业务逻辑，差异仅允许在 CSS 和模板布局层
2. **触控热区**：FAB 56×56dp，手柄 32px，按钮 ≥48dp
3. **拇指热区**：FAB 位于右下角 16dp 边距（绿色区）
4. **动画规范**：Sheet 300ms Standard easing `(0.4, 0.0, 0.2, 1)`，FAB 200ms Bouncy `(0.175, 0.885, 0.32, 1.275)`
5. **字号规范**：Tab 标签 ≥11px
6. **动画实现**：必须用 `transform: translateY()`，禁止 `height/top` 动画
7. **手势隔离**：面板区域 `@touchstart.prevent.stop` 阻止冒泡到地图
8. **安全区域**：`padding-bottom: env(safe-area-inset-bottom, 0px)`
9. **吸附点**：[25vh, 40vh, 80vh]
10. **prefers-reduced-motion**：动画压缩到 0-50ms

### 兼容性约束
- 不破坏 GisMobileNav 对外 props/events 接口（GisData.vue 调用不变）
- 不破坏 MapControlPanel 对外 props/events 接口（MapDrawer.vue 调用不变）
- 不破坏 GisDataPanel 对外 props/events 接口
- 桌面端布局行为保持不变

### 性能约束
- Bottom Sheet 拖拽使用 `requestAnimationFrame`，帧率 60fps
- FAB 展开/收起动画不阻塞主线程
- Tab 切换从 v-if 改 v-show 后，首屏挂载 4 个组件，需评估内存占用（可接受，因状态保留收益 > 内存成本）

### 检验约束（来自 phase1-inspection-rules-v2.md）
- 5×6 测试矩阵：5 断点 × 6 验收点
- L1-L5 五层检验模型
- V3/V4 补充规则：whiteSpace、flex-shrink、canvas 尺寸匹配等
- 弹窗/抽屉激活后必须复检
- DPR 影响与 viewport 设置

5. **修改** `src/components/data/GisDataPanel.vue` —— 复用 useDataPanelTabs

- `src/composables/useBreakpoint.ts` —— 一期已完成 5 级断点，无需改动

### 可复用能力
- `useBreakpoint()` —— 断点检测，FAB 仅在 isMobile 时显示
- `eventBus` —— 地图事件总线，FAB 绘制工具按钮可复用

- `MapControlPanel` 的功能逻辑 —— 城市选择/CRS选择/绘制工具逻辑完整，仅需布局改造

- 二期方案明确要求 Bottom Sheet + FAB + Tab v-show + 共享 composable，这 4 项是核心
- MapControlPanel 的 layout prop 是 FAB 改造的必要配套（移动端控件收起到 FAB）
- GisData.vue 的修改是 FAB 嵌入的必要配套（移动端地图区需放置 FAB）
- 不涉及第三期的手势增强、Web Worker、虚拟滚动
- 不涉及第四期的设计 Token 化
- 不涉及第五期的代码清理和目录重构

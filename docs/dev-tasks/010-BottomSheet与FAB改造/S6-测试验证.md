## 五、二期改造方案调整：移除 FAB（2026-06-25 更新）

### 5.1 用户反馈
用户反馈："FAB效果和结构和整体设计风格完全不符合。点击后的动画和功设计都不行，已经进入数据查看阶段你还弄个fab让我绘制？完全是为了fab来左的。完全没用。"

### 5.2 根因分析
FAB 悬浮按钮承载的是"绘制工具"（点/线/面/清除），但数据查看场景（GisData.vue）的核心任务是**数据浏览与检视**，而非绘制。绘制功能在 MapDrawer.vue（专门的绘制组件）中已有完整实现（MapControlPanel 集成了绘制 dropdown）。FAB 在数据查看场景属于**场景错配**，违反了 Shneiderman 的"任务-工具匹配"原则。

### 5.3 调整决策
**移除数据查看页面的 FAB 组件**，理由：
1. 场景错配：数据查看 ≠ 绘制操作
2. 功能冗余：MapControlPanel 已集成绘制功能
3. 风格冲突：FAB 的 Material Design 风格与 Element Plus 整体风格不统一
4. 空间节约：移除 FAB 后地图区域更完整

## 六、移动端布局溢出修复（2026-06-25 更新）

### 6.1 问题现象
在 390x844 和 430x932 分辨率下，发现两个严重溢出问题：
1. **底部 Tab 栏宽度只有 99px**（4 个 Tab × 24.75px），未占满屏幕宽度
2. **内容区高度为 0**，地图不可见，导航区占满整个剩余空间

### 6.2 根因分析

#### 问题1：Tab 栏宽度不足
- **原因**：`.mobile-nav-area` 使用 `display: flex`（默认 row 方向），子元素 `.gis-mobile-nav` 默认 `flex: 0 1 auto`，宽度按内容计算
- **机制**：`.mobile-tab-bar` 内 4 个 `flex: 1` 的 Tab 项，每个按内容宽度（图标+文字）约 24.75px，总宽 99px
- **修复**：给 `.gis-mobile-nav` 设置 `flex: 0 0 auto` + `width: 100%`，确保占满父容器宽度

#### 问题2：内容区高度为 0
- **原因**：`.gis-mobile-nav` 设置了 `flex: 1 1 100%`，在父级 `.mobile-nav-area`（display:flex, row）中 flex-basis:100% 是宽度，但 `.gis-mobile-nav` 自身是 `display: flex; flex-direction: column`，其父级 `unified-layout` 是 `flex-direction: column`
- **机制**：关键混淆点 —— `.gis-mobile-nav` 的 `flex: 1 1 100%` 作用于 `.mobile-nav-area`（row 容器）中是宽度 100%，没问题；但 `.mobile-nav-area` 在 `unified-layout`（column 容器）中没有固定高度，被 `.gis-mobile-nav` 的内容（BottomSheet teleport 占位）撑开为 796px，挤压了 `content-area`
- **修复**：
  1. `.gis-mobile-nav` 改为 `flex: 0 0 auto`（不扩展不收缩）
  2. `.mobile-nav-area` 设置 `height: calc(56px + env(safe-area-inset-bottom, 0px))` 固定高度 + `flex-grow: 0` + `overflow: hidden`

#### 问题3：Tab 栏高度溢出 1px
- **原因**：`.mobile-tab-bar` 的 `height: 56px` + `border-top: 1px` = 57px（content-box 模型），而 `.mobile-nav-area` 高度 56px
- **修复**：给 `.mobile-tab-bar` 添加 `box-sizing: border-box`

|---|---|---|
| `src/components/data/GisMobileNav.vue` | 修改 | `.gis-mobile-nav` 改为 `flex: 0 0 auto`；`.mobile-tab-bar` 添加 `box-sizing: border-box` |
| `src/components/data/GisData.vue` | 修改 | `.mobile-nav-area` 移动端 media query 添加 `flex-grow: 0`、`height: calc(56px + safe-area)`、`overflow: hidden` |

### 6.5 经验总结：Flex 布局嵌套陷阱

本次修复暴露了 Flex 布局嵌套时的两个隐蔽陷阱，已补充到检验规则：

**陷阱1：flex-basis 百分比的方向歧义**
- `flex: 1 1 100%` 中的 `100%` 是相对于父容器**主轴**方向
- 父容器是 row → 100% 是宽度；父容器是 column → 100% 是高度
- 当组件同时作为不同方向 flex 容器的子项时，容易产生非预期的高度占满

**陷阱2：flex 子项无显式高度被内容撑开**
- 在 column flex 容器中，子项若无 `flex-shrink: 0` + 固定高度，可能被内部 fixed/absolute 元素的占位撑开
- Teleport 组件的挂载点、BottomSheet 的 fixed 遮罩都可能产生不可见的占位空间

**检验规则补充**：
- L1-容器层新增：检查 flex 子项的 `flex-basis` 在不同父容器方向下的实际尺寸
- L1-容器层新增：检查含 Teleport/fixed 子元素的容器是否有显式高度约束
- L5-溢出层新增：验证 column flex 布局中所有子项的高度总和是否等于容器高度

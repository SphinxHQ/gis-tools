### 初步观察（Playwright 实际验证）

**核心问题：GisDataPanel 的 el-tabs 与 GisDataReader 内部的 el-tabs 发生 pane 注册冲突。**

通过 Playwright 实际验证：
1. 加载历史数据后，"数据源"Tab 内容正常显示（GisDataReader 的 el-tabs 可见）
2. 点击"坐标系"Tab 后，Tab 被选中（`[active] [selected]`），但 pane-crs 的 `display: none` + `children: 0` + `innerHTML: 0`
3. 所有 5 个 tab-pane 都处于 `display: none` 状态，内容全部为空
4. el-tabs 的 `v-model` 值正确（`feature`），但 DOM 中没有任何 pane 被激活

**根因**：el-tabs 使用 provide/inject 机制收集子 el-tab-pane。GisDataReader 内部的 `el-tabs type="border-card"` 产生的 4 个 pane 被外层 GisDataPanel 的 el-tabs 错误收集，导致外层 tabs 的 pane 状态完全混乱。

### 其他观察

1. **数据源 Tab 设计不符需求**：用户要求"数据源 + 数据集 1:N"的双层结构（树形/风琴/嵌套卡片），当前只是 GisDataReader 的导入功能，没有数据集列表
2. **要素 Tab 需要还原**：用户要求"还原之前版本的内容"，当前 GisFeatureTree 是从原 GisDataInspactor 拆分的，可能丢失了功能
3. **移动端 GisMobileNav 也有相同问题**：使用 v-if 切换，但同样可能受 el-tabs 嵌套影响

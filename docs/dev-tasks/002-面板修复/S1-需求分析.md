# S1: 需求分析 — 面板修复

## 需求来源

- S0 文档: `S0-原始需求确认.md`
- 已确认需求摘要: 修复 001-布局重构遗留的 el-tabs 嵌套冲突导致左面板 5 个 Tab 全部不可用的问题；重新设计数据源 Tab 为双层结构（数据源 + 数据集 1:N）；确保坐标系/要素/校验/导出 4 个 Tab 功能完整可用；移动端同步修复；S6 必须用 Playwright 实际验证

## 问题描述

001-布局重构任务完成后，左面板（GisDataPanel）的 5 个 Tab（数据源/坐标系/要素/校验/导出）**全部不可用**：

1. **数据源 Tab**：仅作为默认激活 Tab 勉强显示 GisDataReader 内容，但 GisDataReader 内部的 el-tabs 已被外层 el-tabs 错误收集 pane，状态混乱
2. **坐标系/要素/校验/导出 Tab**：点击后 Tab 头部能选中（`[active] [selected]`），但对应 pane 的 `display: none` + `children: 0` + `innerHTML: 0`，内容完全空白
3. **数据源 Tab 设计不符需求**：用户要求"数据源 + 数据集 1:N"双层结构，当前只有导入功能，无数据集列表
4. **移动端 GisMobileNav**：使用 v-if 切换（非 el-tabs），理论上不受嵌套冲突影响，但需实际验证

## 根因分析

### 核心 Bug：el-tabs provide/inject pane 注册冲突

**机制**：Element Plus 的 el-tabs 组件通过 Vue 的 `provide/inject` 机制向后代收集 el-tab-pane 子组件。外层 el-tabs 会 inject 一个 collector，任何后代 el-tab-pane（无论嵌套多深）都会自动注册到最近的祖先 el-tabs。

**冲突路径**：
```
GisDataPanel.vue (el-tabs, 5 panes)
  └─ el-tab-pane[name=source]
       └─ GisDataReader.vue (el-tabs type="border-card", 4 panes)
            ├─ el-tab-pane[name=upload]    ← 被外层 el-tabs 错误收集
            ├─ el-tab-pane[name=text]      ← 被外层 el-tabs 错误收集
            ├─ el-tab-pane[name=draw]      ← 被外层 el-tabs 错误收集
            └─ el-tab-pane[name=history]   ← 被外层 el-tabs 错误收集
```

**结果**：外层 el-tabs 的 pane 列表被污染（预期 5 个，实际 9 个），pane 的 `v-show`/`display` 控制逻辑完全错乱，导致所有 pane 都处于 `display: none` 状态。

**Playwright 验证证据**（S0 阶段）：
- `activeTab` 值正确（如 `'feature'`），Tab 头部 `[active] [selected]` 正确
- 但所有 `.el-tab-pane` 的 `display: none` + `children: 0` + `innerHTML: 0`
- 即使是默认激活的 `source` pane，其内容也因 pane 注册混乱而异常

### 次要问题：数据源 Tab 缺少数据集列表

当前数据源 Tab 仅包含 GisDataReader（导入功能），没有展示已导入的数据集列表。用户要求"数据源 + 数据集 1:N"的双层结构，即：
- 上层：数据源（导入入口）
- 下层：数据集列表（1 个数据源可产生 N 个数据集）

当前数据集切换在顶栏 el-select 中（GisData.vue:158-180），但用户希望数据集列表也在数据源 Tab 内可见，形成完整的"数据源管理"视图。

### 001 任务 S6 漏洞

001 任务的 S6 测试报告（`001-布局重构/S6-测试报告.md`）显示：
- 仅执行了 `npm run build` 构建验证
- 类型检查有预存在错误
- **UI 交互验证全部标注为"待用户确认"**，未实际执行任何浏览器测试
- 结论却标记为"通过"

这是 bug 漏过的直接原因。本次 S6 必须用 Playwright 实际验证每个 Tab 的功能。

1. **不破坏数据流**：useGisDataStore/useMapController/useBreakpoint 不修改
2. **不修改子组件**：GisDataReader/GisDataTransformer/GisFeatureTree/GisDataValidator/GisDataExport 保持不变
3. **不引入新依赖**：仅使用 Element Plus 已有组件（el-segmented/el-collapse/el-card 等）
4. **向后兼容**：GisDataPanel 的 props/emits 接口保持不变，GisData.vue 不需修改
5. **主题适配**：修复后的 UI 需适配深色/浅色主题
6. **S6 必须实际验证**：用 Playwright 逐个验证 5 个 Tab 的功能，不接受仅构建验证

## 验证结论

- **已验证（S0 阶段 Playwright）**：el-tabs 嵌套冲突是根因，所有 pane 的 display:none + children:0
- **已验证（代码审查）**：GisMobileNav 使用 v-if 切换，不涉及 el-tabs 嵌套，理论无此 bug
- **已验证（Element Plus 文档）**：el-segmented 是 Element Plus 2.8+ 组件，支持 v-model + options + block 模式，无 pane 注册机制，适合替代 el-tabs 做面板切换
- **已验证（项目依赖）**：项目已使用 el-segmented（GisDataExport.vue:237, GisFeatureTree.vue:467），组件可用

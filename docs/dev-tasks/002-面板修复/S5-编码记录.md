### M1: 替换 el-tabs 为 el-segmented + v-show

- **文件**: `src/components/data/GisDataPanel.vue`（模板重构）
- **变更**:
  - 移除 `<el-tabs v-model="activeTab" class="panel-tabs" tab-position="top">` 及 5 个 `<el-tab-pane>`

## 偏差记录

### 偏差 D1: GisDataTransformer.vue TDZ 修复

- **偏差类型**: 计划外文件修改（S3 仅规划修改 GisDataPanel.vue）
- **发现时机**: S6 Playwright 测试阶段
- **根因**: `GisDataTransformer.vue` 第 152 行的 `watch(() => props.data, ..., { immediate: true })` 在 setup 阶段同步执行时调用了 `emitActiveDataChange()`，但该函数定义在第 176 行（`const` 声明），处于暂时性死区（TDZ），抛出 `ReferenceError: Cannot access 'emitActiveDataChange' before initialization`。
- **为何 S5 未发现**: 该 bug 是预存缺陷（非本次引入），在 001 任务中因 el-tabs 嵌套冲突导致所有 Tab 无法渲染而被掩盖。本次修复 el-tabs 嵌套后，GisDataTransformer 实际被渲染，TDZ bug 才暴露。
- **处理**: 用户已授权"自行推进，不需要人工确认和干预"。修复方式为将 `emitActiveDataChange` 函数定义移至 `watch` 之前（第 138 行），消除 TDZ。改动范围 1 个函数位置移动，无逻辑变更。
- **影响面**: 仅 `src/components/data/GisDataTransformer.vue` 1 个文件，函数声明位置调整，无 API/接口/数据结构变更。
- **闭环状态**: 已修复，构建通过，S6 Playwright 验证坐标系 Tab 正常渲染。

# S3: 实施计划 — 面板修复

## 修改项清单

### M1: 替换 el-tabs 为 el-segmented + v-show（修复核心 Bug）

- **文件**: `src/components/data/GisDataPanel.vue`（模板部分，当前第 69-121 行）
- **修改内容**:
  1. 移除 `<el-tabs v-model="activeTab" class="panel-tabs" tab-position="top">` 及 5 个 `<el-tab-pane>`
  2. 替换为 `<el-segmented v-model="activeTab" :options="tabOptions" block>` 顶部切换器
  3. 5 个 Tab 内容区改为 `<div v-show="activeTab === 'xxx'">` 平铺布局
  4. 保留所有子组件的 `v-if` 条件（延迟渲染逻辑不变）
  5. 保留所有 props/emits/handlers 逻辑不变
  6. `tabOptions` 定义为 `[{value:'source',label:'数据源'},...,{value:'export',label:'导出'}]`
  7. 禁用状态：坐标系/要素/校验/导出 Tab 在无数据时禁用 → el-segmented 不支持单选项禁用，改为在内容区显示空状态提示
- **影响面**: 仅 GisDataPanel.vue 模板结构，不影响子组件和数据流
- **验证方法**: Playwright 点击每个 Tab，验证内容区可见且有子组件渲染
- **回滚方式**: git checkout GisDataPanel.vue

### M2: 数据源 Tab 双层结构（数据源 + 数据集列表）

- **文件**: `src/components/data/GisDataPanel.vue`（脚本 + 模板）
- **修改内容**:
  1. 脚本：引入 `useGisDataStore`，获取 `datasets`/`activeId`/`setActive`/`removeDataset`
  2. 模板：数据源 Tab 内容区改为 el-collapse 双层结构：
     - 第一层「数据导入」：包含 GisDataReader（上传/文本/绘制/历史）
     - 第二层「数据集 (N)」：数据集卡片列表，每张卡片显示名称、CRS、要素数、顶点数，点击激活，带删除按钮
  3. 数据集卡片点击 → 调用 `setActive(id)`，同步触发父组件数据流
  4. 数据集卡片删除 → 调用 `removeDataset(id)`
- **影响面**: GisDataPanel.vue 脚本新增 store 引用 + 模板新增数据集列表区域
- **验证方法**: Playwright 导入数据后，验证数据源 Tab 内出现数据集卡片；点击卡片切换激活；删除卡片
- **回滚方式**: git checkout GisDataPanel.vue

### M3: 样式适配

- **文件**: `src/components/data/GisDataPanel.vue`（style 部分）
- **修改内容**:
  1. 移除 el-tabs 相关样式（`.panel-tabs :deep(.el-tabs__*)`）
  2. 新增 el-segmented 样式：全宽、小字体、紧凑高度
  3. 新增数据集卡片样式：卡片布局、激活态高亮、删除按钮
  4. 新增 el-collapse 样式适配
  5. 确保深色/浅色主题兼容（使用 CSS 变量）
- **影响面**: 仅样式，无逻辑影响
- **验证方法**: Playwright 截图验证视觉效果 + 深色主题切换
- **回滚方式**: git checkout GisDataPanel.vue

## 风险与回滚总述

**风险**：
- el-segmented 不支持单选项 disabled → 改为内容区空状态提示（低风险）
- 数据集卡片激活态需与顶栏 el-select 同步 → 复用同一 store，天然同步（无风险）
- el-collapse 默认展开状态需合理设置 → 默认两个都展开（低风险）

**回滚**：所有修改集中在 GisDataPanel.vue 单文件，`git checkout src/components/data/GisDataPanel.vue` 即可完整回滚。

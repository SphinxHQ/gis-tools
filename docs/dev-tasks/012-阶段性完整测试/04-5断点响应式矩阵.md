## 四、标准验证流程（来自 912 §4.4）

```
对每个断点（xs → sm → md → lg → xl）执行：

1. resize_page(w, h) 切换视口
2. navigate_page 强制刷新（避免缓存干扰）
3. upload_file 测试数据
4. 等待地图初始化（~2s）
5. 采集关键元素 boundingRect：
   - topBar
   - contentArea
   - mapContainer
   - navArea（移动端）/ leftPanel（桌面端）
6. 验证 document.scrollWidth ≤ document.clientWidth
7. 填写本文件矩阵表
```

### 4.1 验证脚本模板

```javascript
// evaluate_script: 采集关键元素尺寸
() => {
  const els = {
    topBar: '.top-bar',
    content: '.content-area',
    map: '.map-container',
    navArea: '.mobile-nav-area',
    leftPanel: '.left-panel',
  }
  const result = {}
  for (const [name, sel] of Object.entries(els)) {
    const el = document.querySelector(sel)
    if (!el) { result[name] = { exists: false }; continue }
    const rect = el.getBoundingClientRect()
    const display = getComputedStyle(el).display
    result[name] = {
      exists: true,
      display,
      width: Math.round(rect.width),
      height: Math.round(rect.height),
      top: Math.round(rect.top),
      left: Math.round(rect.left),
    }
  }
  result.viewport = {
    scrollWidth: document.scrollWidth,
    clientWidth: document.clientWidth,
    hasHOverflow: document.scrollWidth > document.clientWidth,
    innerWidth: window.innerWidth,
    innerHeight: window.innerHeight,
    devicePixelRatio: window.devicePixelRatio,
  }
  return result
}
```

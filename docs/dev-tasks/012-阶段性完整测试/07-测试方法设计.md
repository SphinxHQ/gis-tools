## 四、执行方法

### 4.1 前置检查流程

```
Step 1: navigate_page(url="http://localhost:5173/gis-tools/")
Step 2: wait_for(selector="body", state="attached")
Step 3: take_snapshot() → 确认页面加载
Step 4: evaluate_script → 检查关键元素是否存在
```

### 4.2 文件上传测试流程

```
Step 1: take_snapshot() → 查找导入按钮 UID
Step 2: click(uid=<导入按钮UID>)
Step 3: take_snapshot() → 确认对话框打开
Step 4: upload_file(uid=<文件输入UID>, filePath="<path/to/1.shp>")
Step 5: wait_for(timeout=2000)
Step 6: take_snapshot() → 验证解析结果
```

### 4.3 Tab 切换测试流程

```
Step 1: take_snapshot() → 查找 Tab 按钮
Step 2: click(uid=<Tab1>)
Step 3: take_snapshot() → 记录当前 Tab
Step 4: click(uid=<Tab2>)
Step 5: take_snapshot() → 验证切换
```

### 4.4 右键菜单测试流程

```
Step 1: evaluate_script → 获取地图要素
Step 2: click(x=<要素x>, y=<要素y>, button="right")
Step 3: take_snapshot() → 确认菜单出现
```

### 4.5 响应式断点测试流程

```
断点序列: [1920, 1280, 1024, 768, 375]
For each breakpoint:
  resize_page(width=<bp>, height=900)
  wait_for(timeout=500)
  evaluate_script → 检查布局 DOM
```

## 五、验收标准

| 类别 | 通过条件 |
|---|---|
| 页面加载 | DOM 加载完成，无 JS 错误 |
| 元素存在 | 导入按钮、Tab、地图容器可见 |
| 文件上传 | 上传后出现要素列表 |
| Tab 切换 | 切换后内容更新 |
| 右键菜单 | 菜单项完整（查看/缩放/删除/导出） |
| 响应式 | 各断点无水平溢出 |

## 六、风险记录

| 风险 | 影响 | 缓解措施 |
|---|---|---|
| 上传路径限制 | 文件上传失败 | 使用 allowlisted 路径 |
| 网络请求超时 | 底图加载超时 | 设置较长 timeout |
| 动态元素 UID 变化 | 点击失效 | 使用相对选择器 |

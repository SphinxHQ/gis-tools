# 12-检查要求补充

> 文档编号：012-12
> 创建日期：2026-06-25
> 目的：补充检查实施过程中的关键约束

---

## 一、MCP 无法验证的操作（禁止执行）

### 1.1 明确禁止的操作

| 操作 | 禁止原因 |
|---|---|
| 点击文件上传 input element | MCP 无法真正验证文件解析结果 |
| 点击 JSON/SHP 导出按钮 | 触发实际下载，无法清理验证 |

### 1.2 允许的操作（其他照常）

- ✅ 文本输入解析（fill + 点击"确认解析"）
- ✅ 点击各种 Tab 切换
- ✅ 点击坐标系/要素/校验/导出 Tab
- ✅ 点击"执行校验"按钮（触发系统提示弹窗）
- ✅ 点击"坐标转换"、"重设坐标系"按钮
- ✅ 点击底图切换（矢量/影像/本地/无）
- ✅ 点击折叠按钮
- ✅ 点击导入按钮（打开对话框）
- ✅ 点击对话框关闭按钮

## 二、场景调整说明

### 2.1 受影响的场景

| 场景 | 调整内容 |
|---|---|
| 场景3 上传文件 Tab | 仅检查 DOM，不实际上传文件 |
| 场景4 文本输入 Tab | 仅检查 DOM，可输入但不实际解析 |
| 场景6 数据加载后 | 通过文本输入方式加载（非文件上传） |
| 场景7 面板 Tab 切换 | 检查导出按钮存在，不点击导出 |

### 2.2 不受影响的场景

| 场景 | 说明 |
|---|---|
| 场景1 页面初始加载 | 无文件操作 |
| 场景2 打开导入对话框 | 仅打开对话框 |
| 场景5 历史记录 Tab | 仅查看表格 |
| 场景8 弹窗与提示 | 仅触发校验弹窗（无文件操作） |

## 三、被禁止的 MCP 操作

```yaml
禁止:
  - upload_file (文件上传)
  - click 文件上传 input element
  - click JSON/SHP 导出按钮 (触发实际下载)
  
允许:
  - navigate_page (页面导航)
  - take_snapshot (获取 DOM)
  - evaluate_script (执行脚本)
  - resize_page (调整视口)
  - list_console_messages (控制台日志)
  - click 所有非文件操作的按钮
  - fill 文本输入框 (用于解析测试)
```

## 四、相关 DOM 元素检查范围调整

### 4.1 仍然检查（仅 DOM 层面）

| 序号 | 元素 | 检查内容 |
|---|---|---|
| 031 | 文件输入框 | 存在性、accept 属性 |
| 030 | 上传区域 | 可见性、文字提示 |
| 022 | 导出选项 | Radio 选项可见性 |
| 023 | 导出编辑器 | Monaco 渲染状态 |
| 024 | 导出按钮 | 按钮存在性 |
| 025 | SHP 导出按钮 | 按钮存在性 |

### 4.2 重点关注项

- 文件输入框的 `accept` 属性是否包含正确的格式（.shp, .geojson, .wkt 等）
- 上传区域的提示文字是否清晰
- 导出按钮的可点击状态（视觉而非实际点击）

## 五、补充检查策略

### 5.1 替代方案

| 原计划 | 替代方案 |
|---|---|
| 上传文件测试解析 | 粘贴文本（GeoJSON 字符串）测试解析 |
| 下载导出文件 | 仅检查按钮存在性与可点击性 |

### 5.2 文本输入解析测试

✅ **允许**：使用 `fill` 工具向文本输入框填充 GeoJSON/WKT 字符串
✅ **允许**：点击"确认解析"按钮
❌ **禁止**：上传实际文件

## 九、发现的问题与修正需求

### 9.1 问题1：场景6采集异常（测试流程问题，非代码问题）

**现象**：场景6在所有 6 个视口下均显示空状态，未出现数据加载后的 left-panel、map-container 等元素。

**原因**：根据 [10-完整检查实施规范.md](./10-完整检查实施规范.md#L296) 的场景6操作定义"关闭对话框，检查数据加载后状态"，采集时仅关闭对话框未执行实际数据导入，`datasets.length` 仍为 0。

**对照证据**：场景7、8 重新执行数据导入后，所有视口均正常显示 left-panel、map-container 等元素。

**结论**：测试流程问题，非代码边界问题，无需修正代码。

### 9.2 问题2：el-tabs__item 在移动端有 -1px 偏移（代码问题，需修正）

**现象**：场景2、3、4、5 在 xs 500×667 和 sm 600×960 视口下，第一个 `el-tabs__item` 的 rect.x=-1, rect.y=-1，所有 `el-tabs__item` 的 rect.y=-1。

**影响范围**：
- 场景2、3、4、5 的移动端视口（xs/sm）
- 共 4 个场景 × 2 个视口 = 8 个数据文件

**原因分析**：
1. Element Plus 的 `el-tabs--border-card` 类型，第一个 `el-tabs__item` 有 `margin-left: -1px` 用来覆盖 header 的左边框
2. `el-tabs__item` 有 `margin-top: -1px` 用来覆盖 header 的上边框
3. 桌面端 dialog 有左右边距（如 1920×1080 下 el-dialog x=513），-1px 偏移被 dialog 内边距吸收
4. 移动端 dialog 全屏（x=0, y=0），-1px 偏移直接导致 Tab 项左上角溢出视窗 1px

**修正方案**：在 [GisDataReader.vue](../../src/components/data/GisDataReader.vue) 的移动端样式中（`@media (max-width: 767px)`），覆盖 `el-tabs__item` 的 margin 为 0。

### 9.3 问题3：el-tabs__item 高度比 header 大 1px（设计行为，无需修正）

**现象**：所有视口下，`el-tabs__header` 高度为 39px，`el-tabs__item` 高度为 40px。

**原因**：Element Plus 的设计，tab 项向下延伸 1px 覆盖 header 的下边框，实现视觉一致性。

**结论**：设计行为，非 Bug，无需修正。

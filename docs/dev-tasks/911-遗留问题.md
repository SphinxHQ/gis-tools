## 五、坐标系规则违反（任务 013）

> 来源：任务 013 S1 根因分析
> 规则基准：`docs/dev-tasks/013-坐标系规则整理/坐标系规则规范.md`（R1~R5）

### 5.1 [✅ 已解决] 重设坐标系违反 R3/R4

| 字段 | 内容 |
|---|---|
| 来源 | 任务 013 S1 根因分析 |
| 规则 | R3（地图坐标系不变性）、R4（重设生成新数据）|
| 现象 | 重设坐标系后，原数据集 crs 被修改 → 地图 projection 跟着变；未生成新数据集 |
| 根因 | `GisDataTransformer.vue` 的 `handleResetCrs` 调用 `updateDataset(activeId, activeVersion.data)` 直接改原数据集 crs |
| 修复 | `handleResetCrs` 改为 `addDataset` 创建同源新数据集，原数据集保持不变；新数据集作为独立节点不继承转换路径 |
| 验证 | 重设后原数据集 crs 不变，新数据集 crs 为目标值，地图投影各自正确 |
| 状态 | ✅ 已解决（任务 013） |

### 5.2 [✅ 已解决] datasetActiveDataMap 缓存陈旧

| 字段 | 内容 |
|---|---|
| 来源 | 任务 013 S1 根因分析 |
| 规则 | R2（数据-坐标系与地图深度绑定）|
| 现象 | 数据集删除/更新时，`datasetActiveDataMap` 缓存未清理 → GisMapSlot 可能渲染到旧 crs 的数据 |
| 根因 | 缓存生命周期未对齐数据集生命周期；`removeDataset` 后缓存未同步清理 |
| 修复 | `GisData.vue` 新增 `watch datasets` 监听 id 数组变化，数据集删除时清理对应缓存 |
| 验证 | 删除数据集后内存中无对应缓存；切换数据集地图投影严格跟随 entry.data.crs |
| 状态 | ✅ 已解决（任务 013） |

### 5.3 [✅ 已解决] GisDataTransformer 版本恢复脆弱

| 字段 | 内容 |
|---|---|
| 来源 | 任务 013 S1 根因分析 |
| 规则 | R2、R3（间接）|
| 现象 | 切换回有转换路径的数据集时，若 `reloadVersionsData` 失败，`activeVersionName` 仍指向 data 为 origin clone 的版本，crs 与 CrsInfo 不一致；`crsVersions[0].data` 引用赋值存在污染风险 |
| 根因 | `reloadVersionsData` 失败未回退 `activeVersionName`；`crsVersions.value[0].data = data` 直接引用赋值 |
| 修复 | `reloadVersionsData` 返回值改为 allSuccess 语义，失败时调用方回退 `activeVersionName` 到 origin；`crsVersions[0].data` 改用 `GisDataInfo.clone(data)` |
| 验证 | 模拟转换失败场景，activeVersion 回退到 origin，显示原数据，地图投影正确 |
| 状态 | ✅ 已解决（任务 013） |

### 5.4 [✅ 已解决] 转换预览污染原数据集地图缓存

| 字段 | 内容 |
|---|---|
| 来源 | 任务 013 验证阶段用户反馈 |
| 规则 | R1（地图随数据集切换）、R3（地图坐标系不变性）|
| 现象 | 读取 4490 数据 → 转 4524 → 切回 4490 数据集，地图 projection 仍为 4524，经纬度坐标显示在顶上一条线 |
| 根因 | `handleActiveDataChange` 在转换预览阶段把转换后数据（crs=4524）写入 `datasetActiveDataMap[原数据集A.id]`；`addDataset` 创建新数据集 B 后未清理 A 的缓存；切回 A 时 `getDatasetActiveData` 返回污染的 4524 数据，而 `GisMapBase` projection 初始化后不可变，导致 4524 坐标在 4490 地图上显示异常 |
| 修复 | 移除 `datasetActiveDataMap` 跨数据集污染机制：`handleActiveDataChange` 不再写入缓存；`GisMapSlot` 的 `:data` 直接绑定 `entry.data`（原始数据）；数据集完全独立隔离，转换后 `addDataset` 自动切换到新数据集，在新地图上显示转换结果 |
| 验证 | typecheck + build 通过；4490→转4524→切回4490，地图正确显示 4490 projection |
| 状态 | ✅ 已解决（任务 013） |

## 需求来源

## 问题描述
项目代码存在 429 个 IDEA 检查问题，本次修复其中 4 类可自动处理的问题。

## 根因分析
- ES6UnusedImports：开发过程中引入但后来未使用的 import 未清理
- JSUnusedLocal/UnnecessaryVar：开发调试遗留或重构后残留
- JSIgnoredPromiseFromCall：异步调用未添加 void/await，可能遗漏错误处理
- CssUnused/Overwritten：CSS 重构后选择器残留，属性声明冲突

## 影响范围
- 仅影响代码清洁度，不影响功能
- 修改文件：约 40-50 个文件（含扩展范围）
- 风险分级：
  - 低风险：删除未使用代码、添加 void、CSS 清理
  - 中风险：ESLint 自动修复（import/order 等）、UMD 改显式 import
  - 需逐个确认：JSUnusedGlobalSymbols（引用链核实）

## 约束条件
- 不修改业务逻辑
- CSS 选择器不处理全局注入内容（:deep()、Element Plus 覆盖等）
- 测试文件中的未使用 import 可能是故意保留的测试数据，需逐个确认
- DuplicatedCode 不处理（用户明确排除）
- GrazieInspection、VueMissingComponentImport 不处理（IDEA 误报）

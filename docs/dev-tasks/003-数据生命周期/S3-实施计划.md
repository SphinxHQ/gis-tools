# S3: 实施计划 — 数据生命周期架构

## 修改项清单

### M1: Store 结构重构

- **文件**: `src/composables/gisDataStore.ts`
- **修改内容**:
  1. `GisDataSetEntry` 接口新增字段：
     - `sourceId: string` — 所属数据源 ID
     - `appendedFrom?: { name: string; count: number }[]` — 追加记录（来源名 + 要素数）
  2. 新增 `GisDataSource` 接口：`{ id: string; name: string; datasetIds: string[] }`
  3. 新增 `activeSourceId` ref — 当前激活的数据源 ID
  4. 新增 `addDataSource(dataInfo: GisDataInfo): string` — 创建新数据源 + 首个数据集，设为激活源
  5. 新增 `updateDataset(id: string, data: GisDataInfo): void` — 更新现有数据集内容（用于"更新当前"）
  6. 修改 `addDataset(dataInfo: GisDataInfo, sourceId?: string): string` — 可选指定源，默认归到 `activeSourceId`
  7. 修改 `appendToDataset(id: string, additional: GisDataInfo): void` — 追加后写入 `appendedFrom` 记录
  8. 新增 `dataSources` computed — 从 `datasets` 派生 `GisDataSource[]` 分组
  9. 修改 `removeDataset` — 删除最后一个数据集时同时删除空数据源
  10. 修改 `clearAll` — 同时清空 `activeSourceId`
- **影响面**: 核心改动，所有 store 消费者受影响（但 API 向后兼容：`addDataset` 无 sourceId 时默认归到当前源）
- **验证方法**: 编译通过 + 导入数据后树形面板显示数据源节点 + CRS 转换后同源出现新数据集
- **回滚方式**: git checkout 该文件

### M2: 导入弹窗改造

- **文件**: `src/components/data/GisDataImportDialog.vue`
- **修改内容**:
  1. 从 store 解构新增 `addDataSource`、`datasets`
  2. 非追加模式：`handleRead` 中 `addDataset(data)` → `addDataSource(data)`
  3. 追加模式：新增目标数据集选择器（`el-select`），列出所有数据集（按数据源分组）
  4. 追加模式 `handleRead`：调用 `appendToDataset(selectedTargetId, data)` 替代 `appendToDataset(activeId, data)`
  5. 追加模式无目标数据集时禁用导入按钮
- **影响面**: 导入流程
- **验证方法**: 导入数据 → 树出现新数据源节点；开追加模式 → 出现目标选择器；选择目标导入 → 目标数据集出现"已追加"标记
- **回滚方式**: git checkout 该文件

### M3: 数据源树形面板适配

- **文件**: `src/components/data/GisDataPanel.vue`
- **修改内容**:
  1. 从 store 解构 `dataSources` 替代 `datasets`
  2. `sourceTreeData` computed 改为从 `dataSources` 派生真实两级树：多数据源根节点 → 各源下数据集
  3. 树节点模板：
     - 数据源节点：FolderOpened 图标 + 名称 + 数据集计数
     - 数据集节点：MapLocation 图标 + 名称 + CRS 标签 + 要素数 + **追加标记**（`appendedFrom` 非空时显示 el-tag "已追加" + tooltip 显示追加详情）
  4. `handleNodeClick` 适配：数据集节点点击激活该数据集并设置 `activeSourceId`
  5. `handleDatasetRemove` 适配：删除后 store 自动处理空数据源
- **影响面**: 数据源 Tab UI
- **验证方法**: 多次导入 → 树显示多个数据源根节点；CRS 转换另存 → 同源下出现新数据集；追加导入 → 目标数据集显示"已追加"标记
- **回滚方式**: git checkout 该文件

### M4: CRS 转换提示更新/另存

- **文件**: `src/components/data/GisDataTransformer.vue`
- **修改内容**:
  1. 从 store 解构 `updateDataset`、`activeId`、`activeSourceId`
  2. `addTransformVersion()` 中 `addDataset(transformedClone)` 替换为调用 `promptUpdateOrSaveAs()`
  3. `handleResetCrs()` 中 `addDataset(resetClone)` 替换为调用 `promptUpdateOrSaveAs()`
  4. 新增 `promptUpdateOrSaveAs(transformedData: GisDataInfo)` 函数：
     - `ElMessageBox.confirm('更新当前数据集还是另存为新数据集？', '变更确认', { confirmButtonText: '另存为新数据集', cancelButtonText: '更新当前数据', ... })`
     - 更新当前 → `updateDataset(activeId, transformedData)`
     - 另存为新数据集 → `addDataset(transformedData, activeSourceId)`（归到当前源）
- **影响面**: CRS 转换/重设流程
- **验证方法**: CRS 转换后弹出提示；选"更新"→ 当前数据集内容变更；选"另存"→ 同源下出现新数据集
- **回滚方式**: git checkout 该文件

### M5: 父组件适配

- **文件**: `src/components/data/GisData.vue`
- **修改内容**:
  1. 从 store 解构 `addDataSource` 替代 `addDataset`
  2. `handleRead()` (line 101): `addDataset(dataInfo)` → `addDataSource(dataInfo)`
  3. `handleDrop()` (line 138): `addDataset(data)` → `addDataSource(data)`
- **影响面**: 导入弹窗读取 + 拖拽导入
- **验证方法**: 通过导入弹窗或拖拽导入数据 → 树出现新数据源节点
- **回滚方式**: git checkout 该文件

### M6: 要素编辑提示更新/另存

- **文件**: `src/components/data/GisFeatureTree.vue`
- **修改内容**:
  1. 引入 `useGisDataStore`，解构 `updateDataset`、`addDataset`、`activeId`、`activeSourceId`
  2. `handleEditorExit()` (line 421) 退出编辑模式时，检查是否有修改（`isInEditMode` 且数据已变更）
  3. 若有变更，调用 `promptUpdateOrSaveAs()`：
     - 更新当前 → `updateDataset(activeId, props.data)`
     - 另存为新数据集 → `addDataset(GisDataInfo.clone(props.data), activeSourceId)`
  4. `handleViewChange` 从 editor 切回 tree 时同样触发提示
- **影响面**: 要素编辑流程
- **验证方法**: 进入要素编辑 → 修改要素 → 退出 → 弹出提示；选"更新"→ 当前数据集更新；选"另存"→ 同源下出现新数据集
- **回滚方式**: git checkout 该文件

## 执行顺序

1. **M1** (store) — 基础，必须先行
2. **M5** (父组件) — 适配新 API，改动最小
3. **M2** (导入弹窗) — 适配新 API + 追加模式增强
4. **M3** (树形面板) — 适配新结构展示
5. **M4** (CRS 转换) — 添加更新/另存提示
6. **M6** (要素编辑) — 添加更新/另存提示

> M2/M3/M4/M5 都依赖 M1，但相互独立。M6 依赖 M1 且涉及编辑流程，放最后。

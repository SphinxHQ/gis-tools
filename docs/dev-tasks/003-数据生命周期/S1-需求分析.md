# S1: 需求分析 — 数据生命周期架构

## 需求来源

- S0 文档: `S0-原始需求确认.md`
- 已确认需求摘要: 将扁平数据集模型重构为"数据源 → 数据集"两级生命周期模型；导入=新数据源；变更操作提示更新/另存；追加模式增强（选择目标+标记+Diff）

## 问题描述

当前 store 是扁平结构 `datasets: GisDataSetEntry[]`，所有数据集平级存放，没有"数据源"这一层概念。导致：

1. 每次导入都创建平级数据集，无法区分"同一数据源的不同版本"与"不同数据源"
2. CRS 转换等变更操作产生的数据集与原数据集平级，无归属关系
3. 追加模式固定追加到当前激活数据集，用户无法选择目标
4. 追加后无标记、无 Diff
5. 变更操作无"更新当前 vs 另存为新数据集"提示

## 根因分析

**根因**：数据模型缺少"数据源"这一层抽象。

- `GisDataSetEntry` 只有 `{id, name, data}`，没有 `sourceId` 归属字段
- `addDataset()` 无条件往扁平数组追加，不区分"新建数据源"与"同源新增数据集"
- 树形面板用假根节点 `数据源 (N)` 视觉模拟，但 store 层无真实层级

**不是症状**：这不是 UI 展示问题，而是数据模型设计问题。仅改 UI 无法解决生命周期归属。

## 现状分析（已有基础设施 vs 缺失部分）

### 已有基础设施

| 能力 | 位置 | 状态 |
|------|------|------|
| 数据集 store | `src/composables/gisDataStore.ts` | 扁平结构，有 add/remove/append/setActive/clearAll |
| `GisDataInfo.clone()` | `src/components/data/GisDataInfo.ts:113-125` | 深拷贝 features + descriptions，可用于"另存" |
| 导入弹窗 + 追加模式开关 | `src/components/data/GisDataImportDialog.vue:16,24-31` | 有追加开关，但追加目标固定为 activeId |
| 数据源树形面板 | `src/components/data/GisDataPanel.vue:86-98` | 假根节点 + 扁平 children，需改为真实层级 |
| CRS 转换创建新数据集 | `src/components/data/GisDataTransformer.vue:210,269` | 002 任务已实现，但创建的是平级数据集 |
| `Common.uuid()` | `src/common/Common.ts` | 可用于生成数据源 ID |

### 缺失部分

1. **数据源抽象**：store 无 `GisDataSource` 类型，无 `addDataSource()` 函数
2. **数据集归属**：`GisDataSetEntry` 无 `sourceId` 字段
3. **追加标记**：`GisDataSetEntry` 无 `appendedFrom` / `hasAppended` 标记字段
4. **目标选择 UI**：导入追加模式无数据集选择器
5. **更新/另存提示**：无通用提示组件或工具函数
6. **Diff 展示**：无追加数据集的 Diff 摘要展示

### 用户场景影响

1. **导入数据**：从"新增平级数据集"变为"新增数据源（含首个数据集）"
2. **CRS 转换/重设**：从"自动创建平级数据集"变为"提示更新当前或另存为新数据集（同源）"
3. **要素编辑**：从"原地编辑无提示"变为"编辑完成后提示更新或另存"
4. **追加导入**：从"追加到当前激活数据集"变为"选择目标数据集追加 + 标记 + Diff"
5. **数据源树**：从"假根 + 扁平列表"变为"多数据源 → 数据集真实层级"

## 约束条件

1. **兼容性**：现有功能（激活/切换/删除/清空）不能破坏
2. **内存态**：本次不涉及持久化，store 重载后数据丢失（与现状一致）
3. **性能**：数据源数量预期 < 20，数据集总数预期 < 100，无需考虑虚拟化
4. **UI 一致性**：树形面板使用 el-tree，提示使用 ElMessageBox（与现有风格一致）
5. **移动端**：数据源树在移动端通过 GisMobileNav 展示，需适配

## 验证结论

- 无外部系统/API 依赖，纯前端 store + UI 改动
- `GisDataInfo.clone()` 已验证可用（002 任务中 CRS 转换已使用）

## 最小改动假设（微创基线）

### 最少需改动的文件/函数

1. **`gisDataStore.ts`**（核心）：
   - `GisDataSetEntry` 加 `sourceId: string` 和 `appendedFrom?: string[]` 字段
   - 新增 `GisDataSource` 接口 `{id, name, datasetIds: string[]}`
   - 新增 `addDataSource(dataInfo): string` — 创建新数据源 + 首个数据集
   - 修改 `addDataset(dataInfo, sourceId?): string` — 可选指定源，默认归到当前激活源
   - 新增 `updateDataset(id, dataInfo): void` — 更新现有数据集内容（用于"更新当前"）
   - 修改 `appendToDataset(id, additional)` — 加 `appendedFrom` 标记
   - 新增 `dataSources` computed — 从扁平 datasets 派生数据源分组

2. **`GisDataImportDialog.vue`**：
   - 非追加模式：调用 `addDataSource(data)` 替代 `addDataset(data)`
   - 追加模式：加目标数据集选择器（el-select），调用 `appendToDataset(targetId, data)`

3. **`GisDataPanel.vue`**：
   - `sourceTreeData` computed 改为从 `dataSources` 派生真实两级树
   - 树节点模板加追加标记（el-tag）

4. **`GisDataTransformer.vue`**：
   - `addTransformVersion()` / `handleResetCrs()` 中 `addDataset()` 改为弹窗提示"更新/另存"
   - 更新 → `updateDataset(activeId, transformedClone)`
   - 另存 → `addDataset(transformedClone, currentSourceId)`

5. **`GisData.vue`**：
   - `handleRead()` / `handleDrop()` 中 `addDataset()` 改为 `addDataSource()`

6. **`GisFeatureTree.vue`**：
   - 编辑退出时提示"更新/另存"（需新增 emit 或回调）

### 明确不需要改动的区域

- `GisDataInfo.ts` — 数据模型类不变
- `GisDataExport.vue` — 只读导出，不变
- `GisDataValidator.vue` — 只读校验，不变
- `GisDataOverview.vue` — 底部状态栏，读 activeDataset，不变
- `GisCrs.ts` — 坐标系类不变
- 地图组件（GisMapTianditu 等）— 不变

### 可复用能力

- `GisDataInfo.clone()` — 用于"另存"时深拷贝
- `Common.uuid()` — 用于生成数据源 ID
- `ElMessageBox.confirm()` — 用于"更新/另存"提示
- `el-tree` — 已用于树形面板，支持多级数据
- `el-select` — 用于追加模式目标选择

### 选择该最小范围的理由

采用"扁平数组 + sourceId 分组字段"而非"嵌套对象数组"结构，因为：
1. 现有 store 函数（find/filter/setActive）大多只需加 sourceId 参数，改动最小
2. `activeDataset` computed 无需改动（仍按 id 查找）
3. 树形面板通过 computed 从扁平数组派生层级，不改变数据存储方式
4. 追加标记只需加字段，不影响现有逻辑

这是能在完整满足需求的前提下改动面最小的方案。

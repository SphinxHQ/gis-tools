# S0: 原始需求确认 — 数据生命周期架构

## 用户原始需求

> 凡是通过导入，导入数据的。都是新增的数据源。数据源内的数据集新增方式，都是在同一份数据上进行转换/编辑等变更操作之后得到的。比如保存json。编辑要素等操作完成后，都要提示 是更新当前的数据，还是另存一份，这种另存的，就是到同一个数据源下面的。从数据生命周期来看。导入是起点，这个入口进去之后的数据变化，都是同一个数据源。再次导入就是新的数据源。另外。导入的时候。有追加模式。追加模式是要选择数据集的。是对某个数据集的追加。并且追加数据之后的数据集，要有标记。表示这是被追加过数据的数据集。在数据的生命周期里面会出现Diff。

## 代码初步定位

### 相关文件

| 文件 | 角色 | 现状 |
|------|------|------|
| `src/composables/gisDataStore.ts` | 数据 store | 扁平结构 `datasets: GisDataSetEntry[]`，无数据源概念；有 `addDataset/removeDataset/appendToDataset/setActive/clearAll` |

| `src/components/data/GisDataPanel.vue` | 数据源树面板 | 用假根节点 `数据源 (N)` 包裹所有扁平数据集，无真实数据源层级 |
| `src/components/data/GisDataTransformer.vue` | CRS 转换 | 转换/重设后调用 `addDataset()` 创建新数据集（002 任务刚加的），但新数据集与原数据集是平级，无同一数据源归属 |
| `src/components/data/GisDataInfo.ts` | 数据模型 | `GisDataInfo { name, originalData, descriptions, features[], crs }`，有 `clone()` 静态方法 |
| `src/components/data/GisDataExport.vue` | 导出 | 导出操作，属于"变更操作"的一种，完成后应提示更新/另存 |
| `src/components/data/GisFeatureTree.vue` | 要素编辑 | 要素编辑入口，编辑完成后应提示更新/另存 |

### 初步观察

1. **Store 是扁平结构**：所有数据集平级存放于 `datasets[]`，没有"数据源"这一层。当前树形面板的"数据源 (N)"根节点是纯视觉假象。
2. **导入即新增数据集**：`addDataset()` 每次都往扁平数组追加，没有"数据源"归属。
3. **追加模式存在但不完整**：追加模式开关已有，但追加目标固定为 `activeId`（当前激活数据集），用户无法选择目标数据集；追加后无标记；无 Diff。
4. **变更操作无更新/另存提示**：CRS 转换（002 已改为创建新数据集）、要素编辑、导出等操作完成后，没有"更新当前数据 vs 另存为新数据集"的提示。
5. **GisDataInfo.clone() 可复用**：深拷贝 features 和 descriptions，适合用于"另存为新数据集"时复制数据。

## AI 复述后的需求

本次任务要将当前的"扁平数据集"模型重构为"数据源 → 数据集"的两级生命周期模型：

1. **数据源（Data Source）是导入的起点**：每次通过导入弹窗导入数据（非追加模式），都创建一个新的数据源。一个数据源下包含一个或多个数据集。
2. **同一数据源内的数据集来自变更操作**：对数据源内某个数据集执行 CRS 转换、要素编辑、保存 JSON 等变更操作后，产生的新数据集归属于同一数据源。
3. **变更操作完成后提示"更新 vs 另存"**：用户执行变更操作后，弹窗提示：
   - "更新当前数据"：用变更结果替换当前数据集的内容
   - "另存为新数据集"：在同一数据源下新增一个数据集
4. **导入追加模式增强**：
   - 追加模式开启时，用户需选择目标数据集（而非自动追加到当前激活数据集）
   - 追加数据后，目标数据集获得"已被追加"标记
   - 被追加的数据集在数据生命周期中展示 Diff（新增了哪些要素）
5. **数据源树形结构真实化**：左侧数据源 Tab 的树形结构改为真实的多数据源 → 数据集层级，每个数据源是独立根节点。

## 需求边界

### 本次包含

1. Store 结构重构：`datasets[]` → `dataSources[]`（每个数据源含 `datasets[]`）
2. 导入逻辑调整：非追加模式 = 新建数据源；追加模式 = 选择目标数据集追加
3. 变更操作（CRS 转换、要素编辑、导出保存等）完成后的"更新 vs 另存"提示
4. 追加数据集的标记（视觉标记 + 数据标记）
5. 被追加数据集的 Diff 展示
6. 数据源树形面板适配新的两级结构
7. 现有功能兼容：激活/切换/删除数据集、清空等操作适配新结构

### 本次不包含

1. 数据源的持久化存储（IndexedDB / localStorage）—— 本次仅内存态
2. 数据源的重命名、合并、拆分等管理操作
3. Diff 的逐要素详细对比视图（仅展示"新增了 N 个要素"级别的摘要）
4. 历史版本回溯 / 撤销栈
5. 跨数据源的数据移动

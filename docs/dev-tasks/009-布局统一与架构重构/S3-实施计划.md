# S3: 实施计划 — 布局统一与架构重构

## 修改项清单

### M1: useBreakpoint.ts 断点体系扩展（4 级 → 5 级）

- 文件: `src/composables/useBreakpoint.ts`
- 修改内容:
  1. `Breakpoint` 类型添加 `'xs'`：`'xl' | 'lg' | 'md' | 'sm'` → `'xl' | 'lg' | 'md' | 'sm' | 'xs'`
  2. `resolveBreakpoint` 函数添加 `<576px → xs` 分支：
     ```typescript
     function resolveBreakpoint(width: number): Breakpoint {
       if (width >= 1280) return 'xl'
       if (width >= 1024) return 'lg'
       if (width >= 768) return 'md'
       if (width >= 576) return 'sm'
       return 'xs'
     }
     ```
  3. `PANEL_WIDTH` / `PANEL_COLLAPSED_WIDTH` 添加 `xs: 0` 键值
  4. `isMobile` 改为 `xs || sm`：`computed(() => globalBreakpoint.value === 'sm' || globalBreakpoint.value === 'xs')`
  5. 新增 `showBottomNav`：`computed(() => globalBreakpoint.value === 'xs' || globalBreakpoint.value === 'sm')`
  6. `showLeftPanel` 保留不变（`panelWidth > 0`，即 md/lg/xl）
  7. `showMobileNav` 保留但改为 `showBottomNav` 的别名（向后兼容 MapDrawer.vue）
- 影响面: 所有使用 `useBreakpoint` 的组件（GisData.vue, MapDrawer.vue）
- 验证方法: 编译通过；`isMobile` 在 <768px 为 true；`showBottomNav` 在 <768px 为 true
- 回滚方式: `git checkout src/composables/useBreakpoint.ts`

### M2: GisData.vue 模板结构统一（双套 DOM → 单套弹性布局）

- 文件: `src/components/data/GisData.vue` L370-430
- 修改内容:
  1. 移除 `<div v-if="showLeftPanel" class="desktop-layout">` 和 `<div v-else class="mobile-layout">` 双套结构
  2. 合并为单一 `unified-layout` DOM：
     ```vue
     <div class="unified-layout">
       <!-- 左面板（CSS 控制 ≥768px 显示） -->
       <div class="left-panel" :class="{ collapsed: panelCollapsed }" :style="{ width: currentPanelWidth + 'px' }">
         <div class="panel-header">...</div>
         <div v-show="!panelCollapsed" class="panel-body">
           <gis-data-panel ... />
         </div>
       </div>
       <!-- 拖拽手柄（CSS 控制 ≥1024px 显示） -->
       <div v-show="!panelCollapsed" class="panel-resize-handle" ... />
       <!-- 主内容区 -->
       <div class="content-area">
         <div class="map-container">
           <gis-map-slot v-for="entry in datasets" ... />
         </div>
         <gis-data-overview :data="activeData" :transform-chain="activeTransformChain" :mode="isMobile ? 'compact' : 'full'" />
       </div>
       <!-- 移动端底部导航（CSS 控制 <768px 显示） -->
       <gis-mobile-nav class="mobile-nav-area" ... />
     </div>
     ```
  3. 地图 `v-for` 从两份合并为一份（当前 desktop-layout 和 mobile-layout 各有一份 GisMapSlot v-for）
  4. `panel-body` 内部 `v-if="!panelCollapsed"` 改为 `v-show="!panelCollapsed"`（避免折叠时销毁组件）
  5. GisDataOverview 的 `mode` 从硬编码 `"full"` / `"compact"` 改为 `:mode="isMobile ? 'compact' : 'full'"`（模板层布局编排）
- 影响面: 主页面布局结构
- 验证方法: 编译通过；375px/768px/1024px/1440px 四个视口下布局完整可用；断点切换时地图不销毁重建
- 回滚方式: `git checkout src/components/data/GisData.vue`

## 风险与回滚总述

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| MapDrawer.vue 的 `isMobile` 语义变化导致行为异常 | 低 | MapDrawer 在移动端行为变化 | MapDrawer 当前可能未被路由使用（需验证）；`xs+sm` 仍表示移动端，语义一致 |
| 统一布局后地图实例在断点切换时仍被销毁 | 低 | 地图状态丢失 | 使用 `v-show` 替代 `v-if`，且地图 v-for 只保留一份 |
| GisMobileNav 在桌面端隐藏但仍挂载，可能影响性能 | 低 | 内存占用增加 | `display: none` 不影响性能；GisMobileNav 内部组件较轻 |
| `el-dialog` 的 `class` 属性可能无法穿透到根元素 | 中 | M5 的 CSS 不生效 | 使用 `:deep()` 或 el-dialog 的 `custom-class` 属性 |

**回滚策略**：每个修改项独立回滚（`git checkout <file>`），M1 回滚后 M2-M4 需要同步回滚（依赖关系）。

## 实施详情

### M1+M2: vite.config.ts 配置
- 在 VitePWA manifest 中添加 share_target 字段
  - action: '/gis-tools/share-receiver'（与部署子路径一致）
  - method: POST, enctype: multipart/form-data
  - files.accept: 声明空间数据文件类型（.geojson/.wkt/.shp/.zip/.dxf/.exf/.txt 等）
- 在 workbox 中添加 importScripts: ['sw-share-target.js']

### M3: public/sw-share-target.js
- 关键修正：文件用独立 Blob Response 存储，避免 JSON.stringify 丢失二进制数据
  - 元数据 → JSON Response → key: /gis-tools/share-pending-meta
  - 每个文件 → 独立 Blob Response → key: /gis-tools/share-pending-file-{index}
- 拦截 POST /gis-tools/share-receiver，读取 formData，暂存到 caches，303 重定向到 /?share=1
- 错误处理：暂存失败重定向到 /?share=error

### M4: src/composables/useShareReceiver.ts
- 检测 ?share=1 或 ?share=error
- 从 Cache Storage 读取元数据 + 文件 Blob
- 复用 SimpleDataFormat.read 解析 + addDataSource 导入 + localDb.add 持久化
- 多文件逐个导入，每个创建独立数据源
- 成功/失败/部分成功三种 ElMessage 提示
- 处理完成后清理缓存 + 清除 URL 参数

### M5: GisData.vue
- import 添加 onMounted 和 useShareReceiver
- onMounted 中调用 initShareReceiver()

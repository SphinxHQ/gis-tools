### M3: 新建 public/sw-share-target.js（SW 拦截逻辑）
- 文件: `public/sw-share-target.js`（新建）
- 修改内容:
  - 注册 fetch 事件监听器
  - 拦截 `url.pathname === '/gis-tools/share-receiver' && request.method === 'POST'`
  - 读取 formData：`files`（多文件）、`title`、`text`、`url`
  - 元数据（title/text/url/timestamp/fileCount/fileNames）存为 JSON Response，key: `/gis-tools/share-pending-meta`
  - 每个文件单独存为 Blob Response，key: `/gis-tools/share-pending-file-{index}`（避免 JSON.stringify 丢失二进制）
  - 303 重定向到 `/gis-tools/?share=1`
  - 错误处理：暂存失败时重定向到 `/?share=error`
- 影响面: 新增 SW 拦截逻辑，不影响现有 workbox 预缓存/运行时缓存
- 验证方法: 移动端分享文件后，DevTools → Application → Cache Storage 应出现 `share-data-cache`，含 meta + file-0 等 entries；URL 应跳转到 `/?share=1`
- 回滚方式: 删除 public/sw-share-target.js + 移除 importScripts 配置

### M4: 新建 src/composables/useShareReceiver.ts（页面读取分享数据）
- 文件: `src/composables/useShareReceiver.ts`（新建）
- 修改内容:
  - 导出 `useShareReceiver()` composable，返回 `init()` 方法
  - `init()`:
    1. 检测 `location.search.includes('share=1')`，不匹配则 return
    2. 打开 `share-data-cache`，读取 `/gis-tools/share-pending-meta`
    3. 按 meta.fileCount 读取每个 `/gis-files/share-pending-file-{i}` 的 Blob
    4. 对每个文件：`new File([blob], name, {type})` → `await blob.arrayBuffer()` → `SimpleDataFormat.read(arrayBuffer)` → `addDataSource(dataInfo)` → `localDb.add(name, arrayBuffer, meta)`
    5. 多文件逐个导入，每个文件创建独立数据源
    6. 成功后清理缓存：删除 meta + 所有 file entries
    7. `history.replaceState(null, '', location.pathname)` 清除 URL 参数
    8. 错误处理：try/catch，失败时 ElMessage.error 提示，仍清理缓存和 URL
  - TypeScript 严格类型：定义 `ShareMeta` 接口（title/text/url/timestamp/fileCount/fileNames: string[]）
- 影响面: 新增 composable，不影响现有组件
- 验证方法: 模拟分享流程后页面加载，应自动出现新数据集；DevTools Console 应打印导入日志；Cache Storage 中 share-data-cache 应被清理；URL 应无 ?share=1
- 回滚方式: 删除 useShareReceiver.ts + 移除 GisData.vue 中的调用

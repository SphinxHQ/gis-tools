### 改动 1: `handleResetCrs` 改为 `addDataset`（修复 R3、R4）

**位置**：[GisDataTransformer.vue:332-371](../../src/components/data/GisDataTransformer.vue)

**改动前**（违反 R3、R4）：
```typescript
const handleResetCrs = (crs: CrsInfo) => {
  resetCrsDialogVisible.value = false
  if (activeVersion.value) {
    activeVersion.value.data.crs = new GisCrs(crs.epsgCode)  // 改活跃版本 data 的 crs
    // ... 更新 descriptions
    if (activeVersion.value.name === 'origin') {
      originData.value.crs = new GisCrs(crs.epsgCode)  // 改 origin

    }

      nextTick(() => { isTransforming.value = false })
    }
  }
}
```

**改动后**（符合 R3、R4）：
```typescript
/**
 * 重设坐标系：基于当前活跃版本数据生成一份声明了新 crs 的新数据集。
 * - 规则 R3：原数据集的 crs 与对应地图 projection 保持不变
 * - 规则 R4：重设必然生成新数据集（同源），原数据集保持不变
 * - 与 addTransformVersion 不同：重设不进行坐标数值转换，仅声明新 crs
 *   （用于修正原数据 crs 声明错误的场景）
 */
const handleResetCrs = (crs: CrsInfo) => {
  resetCrsDialogVisible.value = false
  if (!activeVersion.value) return

  const sourceVer = activeVersion.value
  const sourceData = sourceVer.data
  if (!sourceData) return

  // 1. 深拷贝源数据，避免引用污染
  const resetClone = GisDataInfo.clone(sourceData)
  resetClone.crs = new GisCrs(crs.epsgCode)
  // 更新 descriptions（坐标系元信息）
  if (resetClone.descriptions) {
    const simpleName = crs.name?.split('/')[0].trim() || ''
    let crsLabel = ''
    if (simpleName.includes('2000')) crsLabel = '2000国家大地坐标系'
    else if (simpleName.includes('54')) crsLabel = '54北京坐标系'
    else if (simpleName.includes('80')) crsLabel = '西安80坐标系'
    resetClone.descriptions['坐标系'] = crsLabel
    resetClone.descriptions['几度分带'] = (crs.zoneDegree ?? 0) > 0 ? crs.zoneDegree : ''
    resetClone.descriptions['投影类型'] = CrsCategory.fromProjected(crs.projected) === CrsCategory.Projected ? '高斯克吕格' : ''
    resetClone.descriptions['计量单位'] = CrsCategory.fromProjected(crs.projected) === CrsCategory.Projected ? '米' : '度'
    resetClone.descriptions['带号'] = (crs.zoneNumber ?? 0) > 0 ? String(crs.zoneNumber) : ''
  }
  const sourceName = originData.value?.name || '未命名'
  resetClone.name = `${sourceName} (重设 EPSG:${crs.epsgCode})`

  // 2. 创建新数据集（同源），原数据集保持不变 —— R3、R4
  isTransforming.value = true
  const newDatasetId = addDataset(resetClone, activeSourceId.value ?? undefined)
  nextTick(() => { isTransforming.value = false })

  // 3. 不修改原数据集的 crsVersions / activeVersionName
  //    新数据集作为独立节点，由用户后续按需在其上继续转换
  //    （原数据集的转换路径保持不变，切换回时仍可恢复）
}
```

**关键差异**：
- 不再修改 `activeVersion.value.data.crs`、`originData.value.crs`
- 不再调用 `updateDataset`，改为 `addDataset` 创建同源新数据集
- 不再调用 `saveVersionsToDataset` / `emitActiveDataChange`（新数据集是独立节点，不影响原数据集的转换路径）
- 新数据集切换后由 watch props.data 自然走"无转换路径"分支，重置为 origin

**改动后**：
```typescript
/**
 * 每个数据集的活跃数据（转换后），用于多地图实例渲染。
 * - 规则 R2：地图 projection 直接来源于此活跃数据的 crs.epsgCode
 * - 生命周期必须与数据集对齐：数据集删除时清理缓存，避免 GisMapSlot 渲染到陈旧 crs 数据
 */
const datasetActiveDataMap = ref<Map<string, GisDataInfo>>(new Map())

const getDatasetActiveData = (entryId: string, entryData: GisDataInfo): GisDataInfo => {
  return datasetActiveDataMap.value.get(entryId) ?? entryData
}

// 监听活跃数据集变化
watch(activeDataset, (dataset) => {
  if (dataset) {
    activeData.value = dataset.data
    activeTransformChain.value = dataset.data?.crs?.epsgCode ? [dataset.data.crs.epsgCode] : []
  } else {
    activeData.value = new GisDataInfo()
    activeTransformChain.value = []
  }
}, { immediate: true })

/**
 * 数据集生命周期对齐：数据集被删除时同步清理活跃数据缓存。
 * - 规则 R2：避免缓存陈旧导致 GisMapSlot 渲染到已删除数据集的旧 crs 数据
 * - 仅监听 datasets 长度变化（删除场景），不监听 data 字段变化（避免与 updateDataset 冲突）
 */
watch(() => datasets.value.map(d => d.id), (newIds, oldIds) => {
  if (!oldIds) return
  const removed = oldIds.filter(id => !newIds.includes(id))
  if (removed.length > 0) {
    removed.forEach(id => datasetActiveDataMap.value.delete(id))
  }
}, { deep: true })
```

**关键差异**：
- 新增 watch datasets 监听 id 数组变化，数据集删除时清理对应缓存
- 不监听 data 字段变化（updateDataset 场景由 GisDataTransformer 的 isTransforming 标志位控制，无需在此处同步）

### 改动 3: `reloadVersionsData` 失败回退 + `crsVersions[0].data` 引用隔离（健壮性）

**位置**：[GisDataTransformer.vue:170-182 reloadVersionsData](../../src/components/data/GisDataTransformer.vue) 和 [GisDataTransformer.vue:222-258 watch props.data](../../src/components/data/GisDataTransformer.vue)

**改动 3a: reloadVersionsData 失败回退**

**改动前**：
```typescript
const reloadVersionsData = (): boolean => {
  let lastSuccess = true
  crsVersions.value.forEach((item, idx) => {
    if (idx > 0) {
      item.data = GisDataInfo.clone(originData.value)

```

**改动后**：
```typescript
/**
 * 重新计算所有非 origin 版本的 data（基于 originData 转换）。
 * - 任一版本转换失败时，回退该版本的 data 为 origin 的 clone（crs 仍为目标 crs，但 data 退回原值）
 * - 调用方需根据返回值决定是否回退 activeVersionName
 */
const reloadVersionsData = (): boolean => {
  let allSuccess = true
  crsVersions.value.forEach((item, idx) => {
    if (idx > 0) {
      item.data = GisDataInfo.clone(originData.value)
      const success = transformData(item.data, item.crs)
      if (!success) {
        allSuccess = false
        // 转换失败：data 退回 origin clone（crs 仍是 origin 的 crs，与 item.crs 不一致）
        // 调用方应回退 activeVersionName 到 origin
      }
    }
  })
  return allSuccess
}
```

## 风险评估

| 风险 | 概率 | 影响 | 缓解 |
|------|------|------|------|
| handleResetCrs 改为 addDataset 后，原数据集的转换路径未清理 | 低 | 用户切回原数据集仍看到旧转换路径 | 可接受：原数据集的转换路径是其自身属性，不应被重设影响 |
| watch datasets 监听 id 数组性能 | 低 | datasets 通常 < 10 个 | 可接受：deep watch 仅在 id 变化时触发 |
| reloadVersionsData 失败回退到 origin，用户期望看到失败版本 | 低 | 用户体验 | 可接受：失败版本 data 为 origin clone，crs 与 CrsInfo 不一致，显示会误导用户 |

## 三、技术方案评审

### 3.1 el-tree-v2 兼容性（⚠️ 重点）

**评审结论**：S3 已做 spike 验证，但 GisFeatureTree.vue 现有交互复杂（编辑、定位、删除、属性编辑），el-tree-v2 适配成本可能超预期。

**约束 1**：S5 编码 3.3 时，先评估 el-tree-v2 适配工作量。若 >4 小时或交互逻辑受损，立即切换备选方案（自研虚拟滚动：仅渲染可视区域 + 上下缓冲 5 项），避免陷入 el-tree-v2 API 适配泥潭。

### 3.2 Worker 错误处理

**评审结论**：S3 已考虑错误回退，但未明确错误序列化方案。

**约束 2**：Worker 中 GisError 需序列化为 plain object 传输，主线程接收后重建为 GisError 实例。序列化函数需在 `useDataParser.ts` 中实现，确保错误码（GisErrorCode）和消息完整保留。

### 3.3 触屏手势冲突

**评审结论**：S3 方案合理（手柄 `@touchstart.prevent.stop` + 拖拽中禁用地图手势）。

**补充**：3.5 左滑需与列表垂直滚动区分——`touchmove` 水平位移 > 垂直位移时才触发左滑，否则放行垂直滚动。

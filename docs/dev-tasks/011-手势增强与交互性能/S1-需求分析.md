# S1: 需求分析 — 三期：手势增强与交互性能

> 评估基准：`geo-compacter.md` V2.0 §11.3 防返工自检清单
> 方法论：方案设计阶段自检 D1-D5（基于 010-经验教训.md）

## 一、需求拆解与自检

### 1.1 面板拖拽 resize 触控支持（诊断 3.1）

**现状**：`GisData.vue` L86-109 `startResize` 仅处理 `MouseEvent`，无 touch 事件。`@mousedown="startResize"` 绑定在手柄上。

**D1 场景价值自检**：✅ 通过
- 平板端（iPad/Surface）是面板拖拽的核心场景，触控支持不可替代
- 桌面端已有鼠标拖拽，触控补充不冗余

**D2 功能冗余自检**：✅ 通过
- 现有仅鼠标拖拽，触控为新增能力，非重复

**D3 风格一致性自检**：✅ 通过
- 触摸事件与 mouse 事件成对出现，符合 geo-compacter.md §代码输出规范

**D4 用户路径自检**：✅ 通过
- 面板宽度调整服务于数据浏览核心任务（左侧面板看数据列表，右侧看地图）

**结论**：✅ 纳入三期，优先级 🟡 中

**改造要点**：
- `startResize` 增加 `TouchEvent` 重载，`@mousedown` 改为 `@mousedown + @touchstart`
- `onMouseMove` / `onMouseUp` 增加触屏版（`touchmove` / `touchend`）
- 使用 `passive: false` 处理 `touchmove` 防止页面滚动
- 注意：`e.clientX` 在 TouchEvent 中为 `e.touches[0].clientX`

---

### 1.2 Web Worker 解析大文件（诊断 3.2）

**现状**：`DataFormat.ts` 所有解析在主线程执行，>5MB 文件阻塞 UI。

**D1 场景价值自检**：✅ 通过
- 大文件（>5MB）解析阻塞 UI 是真实性能痛点，Web Worker 不可替代（主线程无法既渲染又解析）

**D2 功能冗余自检**：✅ 通过
- Web Worker 是对现有 DataFormat 的性能优化路径，非功能重复
- 复用现有解析逻辑，仅切换执行线程

**D3 风格一致性自检**：✅ 通过
- Vite 原生支持 Web Worker（`new Worker(new URL('./worker.ts', import.meta.url), { type: 'module' })`）

**D4 用户路径自检**：✅ 通过
- 大文件导入是数据查看的前置路径，解析性能直接影响用户能否查看数据

**D5 推迟项归属自检**：✅ 通过
- 本期完成 Web Worker 基础设施，>2000 要素 WebGL 渲染推迟至第五期（明确归属）

**结论**：✅ 纳入三期，优先级 🟡 中

**改造要点**：
- 新建 `src/workers/dataParser.worker.ts`：接收 ArrayBuffer + 格式类型，返回 GisDataInfo
- 新建 `src/composables/useDataParser.ts`：>5MB 自动切换 Worker，<5MB 主线程直接解析
- Worker 内复用 DataFormat 的解析函数（需确保解析逻辑可序列化，无 DOM 依赖）
- 进度回调：Worker `postMessage` 进度百分比，UI 显示进度条
- 错误处理：Worker `onerror` 回退到主线程解析

---

### 1.3 要素列表虚拟滚动（诊断 3.3）

**现状**：`GisFeatureTree.vue` 直接渲染所有要素到 DOM，>500 个性能下降。

**D1 场景价值自检**：✅ 通过
- >500 要素列表渲染卡顿是真实痛点，虚拟滚动不可替代

**D2 功能冗余自检**：✅ 通过
- 虚拟滚动是对现有列表的性能优化，非功能重复

**D3 风格一致性自检**：✅ 通过
- `el-table-v2` / `el-tree-v2` 是 Element Plus 官方虚拟滚动方案，风格一致

**D4 用户路径自检**：✅ 通过
- 要素列表浏览是数据检视的核心路径

**结论**：✅ 纳入三期，优先级 🟡 中

**改造要点**：
- 阈值判定：要素 >500 切换到虚拟滚动组件，≤500 保持现有 el-tree
- 需评估：el-tree-v2 是否支持现有的要素类型筛选、属性展示、点击定位地图功能
- 风险：el-tree-v2 API 与 el-tree 差异较大，可能需要重构交互逻辑
- 备选方案：若 el-tree-v2 兼容性差，考虑自研虚拟滚动（仅渲染可视区域 + 上下缓冲）

---

### 1.4 长按手势选点（诊断 3.4）⚠️ 重点自检

**现状**：地图无长按选点弹出气泡功能。

**D1 场景价值自检**：⚠️ 需评估
- **关键问题**：数据查看场景（GisData.vue）的核心任务是"浏览检视已有数据"，而非"采集新点位"
- 地图已有单击选择要素功能，长按选点（任意位置）与数据查看场景的核心任务关联性弱
- 长按选点更适用于"数据采集/编辑"场景，而非"数据查看"场景

**D2 功能冗余自检**：⚠️ 部分冗余
- 单击选择要素 vs 长按选点：单击选已有要素，长按选任意位置——功能有差异，但在数据查看场景下，用户更需要选要素而非选空地点

**D4 用户路径自检**：❌ 不通过
- 数据查看场景核心路径：导入数据 → 浏览要素列表 → 点击要素定位地图 → 查看属性
- 长按选点不在该核心路径上，属于"采集"路径，场景错配风险高（⚠️ 二期 FAB 教训）

**结论**：❌ **建议从三期移除，推迟至数据编辑场景改造**

**理由**（基于二期 FAB 返工教训 D1/D4）：
1. 场景错配：长按选点服务于"采集"场景，数据查看场景需要的是"选择已有要素"
2. 功能冗余：单击选要素已覆盖数据查看场景的选点需求
3. 用户路径：长按选点不在数据查看核心路径上

**替代方案**：三期改为增强"要素选择"交互——长按要素显示属性气泡（与单击选要素互补，非选空地）

---

### 1.5 列表项左滑操作（诊断 3.5）⚠️ 重点自检

**现状**：数据集卡片在 `GisData.vue` L333 已有删除按钮（`dataset-card-delete`），点击 `removeDataset(id)` 删除。

**D1 场景价值自检**：✅ 通过（移动端专属）
- 桌面端已有删除按钮，左滑作为**移动端专属快捷方式**，提供增量价值
- 移动端单手操作时，左滑比精准点击小图标更易触达

**D2 功能冗余自检**：⚠️ 部分冗余但可接受
- 桌面端保留删除按钮，移动端增加左滑 → 非完全冗余，是平台差异化交互
- 关键约束：左滑仅移动端启用（`@media max-width: 767px`），桌面端不显示左滑提示

**D4 用户路径自检**：✅ 通过
- 数据集管理（删除）是数据查看场景的辅助路径，左滑加速操作

**结论**：✅ 纳入三期，优先级 🟢 低（移动端增强项）

**改造要点**：
- `GisDataOverview.vue` 或 `GisData.vue` 卡片新增左滑手势（`@touchstart` / `@touchmove`）
- 左滑 48dp 显示删除按钮，动画 200ms Standard easing
- 仅移动端启用（CSS `@media max-width: 767px` 控制提示显隐）
- 桌面端保留现有删除按钮不变

## 二、优先级排序与风险评估

### 2.1 优先级矩阵

| 改造项 | 场景价值 | 风险 | 优先级 | 建议 |
|---|---|---|---|---|
| 3.1 面板拖拽触控 | 高 | 低（触屏事件成对添加） | P0 | ✅ 纳入 |
| 3.2 Web Worker 大文件 | 高 | 中（Worker 序列化、进度回调） | P0 | ✅ 纳入 |
| 3.3 虚拟滚动 | 高 | 中高（el-tree-v2 兼容性） | P1 | ✅ 纳入（需备选方案） |
| 3.4 长按选点 | 低（场景错配） | 低 | - | ❌ 移除（推迟至编辑场景） |
| 3.5 左滑操作 | 中（移动端专属） | 低 | P2 | ✅ 纳入（移动端快捷方式） |

### 2.2 风险评估

| 风险 | 概率 | 影响 | 缓解措施 |
|---|---|---|---|
| el-tree-v2 与现有交互不兼容 | 中 | 高 | S3 前做技术验证（spike），备选自研虚拟滚动 |
| Worker 解析逻辑无法序列化 | 低 | 高 | 确保解析函数纯函数化，无 DOM/全局状态依赖 |
| 触屏拖拽与地图手势冲突 | 中 | 中 | 手柄区域 `@touchstart.prevent.stop`，拖拽中禁用地图手势 |
| 性能验证无量化工具 | 中 | 中 | S6 使用 Performance API + FPS 监控脚本 |

## 五、经验教训应用总结

本 S1 严格应用 geo-compacter.md §11.3.1 方案设计阶段自检：

| 自检项 | 应用结果 |
|---|---|
| D1 场景价值 | 3.4 长按选点未通过（场景错配），建议移除 |
| D2 功能冗余 | 3.4 与单击选要素部分冗余 |
| D3 风格一致性 | 3.3 虚拟滚动用 el-tree-v2，与 Element Plus 一致 |
| D4 用户路径 | 3.4 不在数据查看核心路径（⚠️ 二期 FAB 教训） |
| D5 推迟项归属 | WebGL 渲染明确推迟至第五期 |

**关键决策**：基于二期 FAB 返工教训（场景优先于规范），主动识别并移除 3.4 长按选点（场景错配），避免重蹈"为用规范而用规范"的覆辙。

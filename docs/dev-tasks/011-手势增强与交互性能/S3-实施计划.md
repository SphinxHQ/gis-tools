> 基准：S2 评审决议 + geo-compacter.md §11.3 自检清单

## 一、el-tree-v2 技术验证（S2 前置任务）✅

**验证结论**：
- Element Plus 2.8.1 包含 `ElTreeV2`（虚拟滚动树），API 与 `ElTree` 大部分兼容
- `el-tree-v2` 要求：必须设置 `key` 属性，数据结构为扁平数组 + `level` 字段（非嵌套）
- **GisFeatureTree.vue 现状**：按几何类型分组（Point/LineString/Polygon），树深度 2 层
- **兼容性评估**：现有树结构需转换为 el-tree-v2 的扁平数组格式，交互逻辑（选中/定位/删除）可保留
- **决策**：采用 el-tree-v2，S5 编码时封装适配层；若适配成本过高，备选自研虚拟滚动（仅渲染可视区域 + 上下缓冲 5 项）

## 三、技术方案细节

### 3.0 设备能力检测（P0 前置）

**接口定义**：
```typescript
export interface DeviceCapabilities {
  /** 触摸点数（0=无触摸） */
  maxTouchPoints: number
  /** 是否触摸设备 */
  hasTouch: boolean
  /** 指针精度：fine(鼠标)/coarse(手指)/none */
  pointer: 'fine' | 'coarse' | 'none'
  /** 是否支持悬停 */
  hasHover: boolean
  /** 是否偏好减少动效 */
  prefersReducedMotion: boolean
  /** 设备内存(GB)，可能 undefined */
  deviceMemory?: number
  /** CPU 核心数 */
  hardwareConcurrency: number
  /** 网络类型：4g/3g/2g/slow-2g，可能 undefined */
  effectiveType?: string
}
```

**实现要点**：
- 全局单例（类似 useBreakpoint），`matchMedia` 监听变化
- SSR 安全（`typeof window` 检查）
- 提供 `useDeviceCapabilities()` composable，返回响应式 `capabilities`

### 3.1 面板拖拽触控（P0）

**GisData.vue `startResize` 改造**：
```typescript
// 现状：仅 MouseEvent
function startResize(e: MouseEvent) { ... }

// 改造：支持 MouseEvent + TouchEvent
function startResize(e: MouseEvent | TouchEvent) {
  const clientX = e instanceof TouchEvent ? e.touches[0].clientX : e.clientX
  // ... 后续逻辑复用
  window.addEventListener('mousemove', onMove)
  window.addEventListener('mouseup', onUp)
  window.addEventListener('touchmove', onMove, { passive: false })  // prevent 滚动
  window.addEventListener('touchend', onUp)
}
```

**事件隔离**：手柄区域 `@touchstart.prevent.stop`，拖拽中临时禁用地图手势（`map.interactions` 暂停）

### 3.2 Web Worker 大文件解析（P0）

**阈值策略**：
- `content.byteLength > 5MB` 或 `navigator.connection?.effectiveType === '4g'` 且 > 2MB → Worker
- 否则 → 主线程直接解析（DataFormat.ts 现有逻辑）

**Worker 接口**：
```typescript
// dataParser.worker.ts
self.onmessage = async (e: MessageEvent<{buffer: ArrayBuffer, type: GisDataType}>) => {
  const { buffer, type } = e.data
  try {
    const format = getFormatByType(type)  // 复用 DataFormat.ts 逻辑
    const dataInfo = await format.read(buffer)
    ;(self as any).postMessage({ type: 'success', dataInfo })
  } catch (err) {
    ;(self as any).postMessage({ type: 'error', error: serializeError(err) })
  }
}
```

**useDataParser.ts**：
```typescript
export async function parseData(content: ArrayBuffer | string, type?: GisDataType): Promise<GisDataInfo> {
  const size = content instanceof ArrayBuffer ? content.byteLength : content.length
  if (size > 5 * 1024 * 1024 && typeof Worker !== 'undefined') {
    return parseInWorker(content, type)  // Worker 路径
  }
  return parseInMain(content, type)  // 主线程路径（现有逻辑）
}
```

**纯函数化要求**：DataFormat.ts 的 `read` 函数需确保无 DOM/全局状态依赖，可被 Worker 调用

## 原始需求

### 三期核心目标
完善手势词库，实现面板拖拽 resize 的触控支持，引入 Web Worker 解析大文件，虚拟滚动优化要素列表。

### 诊断依据（来自 ux-renovation-plan.md §3）

| # | 违规项 | 位置 | 违反理论 | 严重度 |
|---|--------|------|----------|--------|
| 3.1 | 面板拖拽仅支持鼠标：`startResize` 仅处理 MouseEvent，无 touch 事件 | `GisData.vue` | 触控交互规范 | 🟡 |
| 3.2 | 无 Web Worker 解析大文件：所有解析在主线程，>5MB 阻塞 UI | `DataFormat.ts` | 性能预算规范 | 🟡 |
| 3.3 | 要素列表无虚拟滚动：GisFeatureTree 直接渲染所有要素，>500 性能下降 | `GisFeatureTree.vue` | 性能预算规范 | 🟡 |
| 3.4 | 长按手势缺失：地图无长按选点弹出气泡 | 全局 | Clark 手势词汇表 | 🟡 |
| 3.5 | 列表项左滑操作缺失：数据集卡片无左滑删除/导出 | 全局 | Clark 手势词汇表 | 🟢 |

### 三期改造方案（来自 ux-renovation-plan.md §3）

| 文件 | 操作 | 说明 |
|------|------|------|
| `src/components/data/GisData.vue` | 修改 | `startResize` 增加 touch 事件处理 |
| `src/composables/useDataParser.ts` | 新建 | Web Worker 封装：>5MB 自动切换 Worker |
| `src/workers/dataParser.worker.ts` | 新建 | Web Worker 入口 |
| `src/components/data/GisFeatureTree.vue` | 修改 | 要素 >500 切换 el-table-v2 虚拟滚动；新增左滑手势 |
| `src/composables/useMapGestures.ts` | 新建 | 地图手势增强：双击放大、长按选点+气泡+触觉 |
| `src/components/data/GisDataOverview.vue` | 修改 | 数据集卡片新增左滑操作 |

### 三期验收清单（来自 ux-renovation-plan.md §3）
- [ ] 面板拖拽在触控设备上可用（iPad / Surface）
- [ ] 5MB 文件解析不阻塞 UI（主线程 FPS ≥ 50）
- [ ] 1000 个要素列表渲染 < 1s，滚动帧率 ≥ 55fps
- [ ] 地图双击放大 1.5x，动画 300ms Sharp easing
- [ ] 长按 500ms 弹出气泡，伴随 `navigator.vibrate(10)`
- [ ] 左滑 48dp 显示操作按钮，动画 200ms Standard easing
- [ ] 1000 个要素地图渲染 < 1s

## 经验教训应用要求（基于 010-经验教训.md §五 + geo-compacter.md §11.3）

三期执行必须遵循以下硬性约束：

### 方案设计阶段（D1-D5 自检）
- **D1 场景价值自检**：每个新增手势是否提供不可替代价值？
- **D2 功能冗余自检**：Web Worker 是否与现有 DataFormat 解析逻辑冗余？
- **D3 风格一致性自检**：虚拟滚动是否与 GisFeatureTree 现有交互一致？
- **D4 用户路径自检**：手势是否服务于核心任务（数据浏览/检视）？
- **D5 推迟项归属自检**：本期推迟项是否明确归属期次？

### 执行实施阶段（E1-E8 自检）
- 触摸事件必须与 mouse 事件成对出现（E3/E4 适用）
- Flex 布局遵循"显式约束 + overflow + border-box"三原则
- 方案实施完成后对照组件变更表逐项核对（E7）

### 验证阶段（V1-V6 自检）
- 5 断点全覆盖动态验证（V1/V2/V3）
- 性能用 Performance API / FPS 监控验证（V5/V6）
- 触摸手势标注"待真机验证"（V4）

### 跨平台阶段（P1-P3 自检）
- 提交时使用 `git commit -F` + 临时文件（P2）
- 合并使用 `--no-ff` 保留合并记录（P3）
- 合并后 diff 验证完整性（E1）+ 编译验证（E2）

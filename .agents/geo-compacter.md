## 理论基石与方法论

你基于以下经典理论体系指导设计决策。每次方案输出时，须明确标注所依据的理论，确保设计有理有据、可追溯。

### 1. 信息可视化与地图认知

| 理论来源 | 核心概念 | 在本项目中的应用 |
|----------|----------|------------------|
| **Edward Tufte**《The Visual Display of Quantitative Information》 | **数据墨水比（Data-Ink Ratio）**：最大化数据墨水、最小化非数据墨水；**图表垃圾（Chartjunk）**：消除所有无关装饰 | 地图控件悬浮层设计：最小化 UI 元素面积，将最大像素留给地图数据本身；底图切换器、工具栏均采用半透明毛玻璃背景，避免遮挡地理信息 |
| **Ben Shneiderman**《Visual Information Seeking Mantra》 | **Overview first, zoom and filter, then details-on-demand**：先全貌，再缩放过滤，最后按需展示细节 | 数据集管理流程：先展示数据集概览卡片（Overview）→ 点击后进入要素树按类型筛选（Zoom & Filter）→ 选中要素进入编辑器查看属性（Details-on-demand） |
| **Colin Ware**《Information Visualization: Perception for Design》 | **前注意处理（Pre-attentive Processing）**：颜色、方向、大小、形状在 200ms 内被下意识感知；**格式塔定律（Gestalt Laws）**：邻近性、相似性、闭合性、连续性、共同命运 | 要素渲染分层：点要素用高饱和蓝色圆形（颜色+形状双通道），线要素用绿色（颜色通道），面要素用橙色半透明填充（颜色+面积通道）；转换路径图用贝塞尔曲线的连续性（Gestalt 连续性）表达数据流向 |
| **Alan MacEachren**《How Maps Work》 | **视觉变量（Visual Variables）**：位置、大小、形状、方向、颜色、纹理、透明度在制图中的语义映射 | 点=蓝色 `#2563eb` r6、线=绿色 `#16a34a` w2、面=橙色 `#ea580c` 填充 0.5 透明度 w2 边框——每种几何类型独占一个颜色通道，避免混淆 |
| **Jacques Bertin**《Semiology of Graphics》 | **视网膜变量（Retinal Variables）**：关联性（≈）、选择性（≠）、有序性（O）、数量性（Q）四种感知维度 | CRS 类型标签：地理坐标系用青色（关联性），投影坐标系用紫色（区分性）；要素数量用尺寸梯度（数量性） |

### 2. 交互设计与认知心理学

| 理论来源 | 核心概念 | 在本项目中的应用 |
|----------|----------|------------------|
| **Don Norman**《The Design of Everyday Things》 | **可供性（Affordance）**：物体属性暗示使用方式；**意符（Signifier）**：传达可供性的信号；**映射（Mapping）**：控制与效果的关系；**反馈（Feedback）**：操作后的即时响应；**概念模型（Conceptual Model）**：用户心智模型；**执行与评估鸿沟（Gulfs of Execution & Evaluation）** | 底部 Sheet 手柄（意符：可拖拽）、拖拽时实时跟随（反馈 < 16ms）、松手自动吸附半屏/全屏（映射关系清晰）；要素编辑器 Update/Restore 按钮（缩小执行鸿沟）；坐标转换后自动创建新数据集（缩小评估鸿沟，用户无需猜测转换结果） |
| **Fitts's Law**：MT = a + b · log₂(D/W + 1) | 移动时间（MT）与目标距离（D）成正比、与目标宽度（W）成反比。目标越大越近，操作越快 | 所有触控热区 >= 48dp（增大 W）；高频操作（导入、切换数据集）放在拇指热区底部（缩短 D）；数据集切换入口放在顶部栏（低频操作，允许较远距离） |
| **Hick's Law**：T = b · log₂(n + 1) | 决策时间与选项数量的对数成正比。选项越多，决策越慢 | 底部 Tab 栏不超过 5 个选项；每层菜单选项不超过 7 个；每个 Tab 内部按优先级排序，减少用户扫描成本 |
| **Miller's Law**：7±2 组块 | 人类工作记忆容量约 7±2 个信息组块 | 数据集列表分组展示（按数据源分组，每组内数据集卡片 <= 5 个）；要素树每层节点建议 <= 7 个，超出则折叠或分组 |
| **Steven Hoober**《Designing for Touch》 | **拇指热区图（Thumb Zone）**：单手操作时，屏幕底部 1/3 为"自然区"（绿色），中部 1/3 为"伸展区"（橙色），顶部 1/3 为"困难区"（红色）；**握持模式**：单手（49%）、摇篮式（36%）、双手（15%）| 底部 Tab 栏 56px 位于拇指自然区（绿色）；顶部栏仅放低频率操作（导入、主题切换）；地图 FAB 位于右下角绿色区；底部 Sheet 手柄 32px 位于绿色区 |
| **Josh Clark**《Designing for Touch》 | **手势词汇表（Gesture Vocabulary）**：单指=平移/选择，双指=缩放/旋转，长按=上下文菜单，双击=放大，滑动=删除/操作 | 地图区域：单指平移、双指缩放/旋转、长按 500ms 选点并弹出气泡；面板区域：单指上下滑动切换高度；列表项：左滑显示操作按钮 |

## 设计原则与量化约定

### 1. 响应式策略：Adaptive First

**理念**：遵循 Luke Wroblewski 的 Mobile First 渐进增强。核心功能在移动端和桌面端行为一致，增强功能仅在大屏启用。

**断点体系**（基于主流设备分布）：

| 断点名称 | 视口范围 | 典型设备 | 布局策略 |
|----------|----------|----------|----------|
| `xs`（移动竖屏） | < 576px | iPhone SE/12/13/14/15 (375-430px) | 地图全屏 + 底部 Sheet + FAB 悬浮按钮 |
| `sm`（移动横屏 / 大屏手机） | 576-767px | iPhone 14 Pro Max 横屏 (430px) | 地图全屏 + 可收起的底部 Sheet |
| `md`（平板竖屏） | 768-1023px | iPad Mini (768px), iPad (810px) | 地图为主 + 可折叠左侧面板（默认折叠） |
| `lg`（平板横屏 / 小桌面） | 1024-1279px | iPad Pro 横屏 (1024px) | 左侧面板 + 地图 + 底部状态栏，面板可拖拽 |
| `xl`（桌面大屏） | >= 1280px | 标准显示器 (1440-1920px) | 左侧面板（默认 420px） + 地图 + 底部状态栏 |

**核心原则：单一代码路径（Single Code Path）**

移动端与桌面端的业务逻辑、状态管理、数据处理必须共享同一套代码，绝不允许通过 `if (isMobile)` 分支来编写两套不同的行为逻辑。其理论依据来自：

| 理论来源 | 核心概念 | 在本项目中的应用 |
|----------|----------|------------------|
| **DRY 原则**（Don't Repeat Yourself） | 每一条知识在系统内必须拥有单一、明确、权威的表示 | 坐标系转换、要素校验、数据导出等核心业务逻辑，无论移动端还是桌面端，调用的是完全相同的函数和 composable |
| **Wroblewski**《Mobile First》 | **渐进增强（Progressive Enhancement）**：基础体验面向移动端，大屏逐步增强 | 核心功能层（数据解析、CRS 转换、要素渲染）是移动端和桌面端的公共基线；增强功能（面板 resize、多窗口）仅在大屏通过 CSS 媒体查询启用，而非通过 JS 分支 |
| **Frost**《Atomic Design》 | **模板层是唯一的差异点**：原子、分子、有机体三层的组件行为必须跨端一致，仅模板层（布局编排）允许响应式差异 | GisDataPanel（有机体）在桌面端嵌入左侧面板、在移动端嵌入底部 Sheet，但组件自身的 props/events/内部逻辑完全不变——差异仅由父级模板的 CSS 媒体查询实现 |

**硬性约束**：
- 禁止在组件内硬编码断点数值，统一使用 `useBreakpoint()` 的 `isMobile` / `isTablet` / `isDesktop` computed
- 禁止使用 `v-if` 硬切换两套完全不同的布局结构，改用 CSS `@media` + `flexbox` / `grid` 渐进适配
- 每个断点内的布局必须是 **同一套 DOM 结构**，仅通过 CSS 改变位置、尺寸、可见性
- **禁止在 `<script>` 中编写 `if (isMobile.value) { ... } else { ... }` 来分支业务逻辑**——业务逻辑必须跨端一致，差异仅允许存在于 CSS 层和模板层的布局编排

### 2. 触控交互量化规范

**依据**：Apple HIG (iOS 18)、Material Design 3 (Android)、Hoober《Designing for Touch》

#### 2.1 触控热区

| 元素类型 | 最小热区 (dp) | 推荐热区 (dp) | 最小间距 (dp) |
|----------|--------------|--------------|--------------|
| 地图上悬浮按钮 | 48 x 48 | 56 x 56 | 16 |
| 底部 Tab 栏图标 | 48 x 48 | 48 x 48（整栏高度 56） | 8 |
| 列表行 / 卡片 | 48 高 | 56 高 | 0（连续排列） |
| 弹窗按钮 | 44 x 44 | 48 x 48 | 12 |
| 拖拽手柄 | 48 x 32 | 全宽 x 32 | 0 |
| 表单输入框 | 48 高 | 56 高 | 16 |
| 复选框 / 单选框 | 44 x 44 | 48 x 48 | 8 |

**硬性约束**：任何可点击元素的 CSS 尺寸 < 44px 时，必须通过 `padding` 或伪元素扩展到 >= 44px 的实际热区。视觉尺寸可小于热区。

#### 2.2 拇指热区映射

基于 Hoober 的研究（1,333 人观察数据），单手操作时：

```
┌──────────────────────────────┐
│  🔴 红色区（困难区）          │  ← 顶部 1/3：仅放低频率操作
│  - 顶栏（仅放标题/Logo/主题）│     单手操作需要调整握持姿势
├──────────────────────────────┤
│  🟡 橙色区（伸展区）          │  ← 中部 1/3：次级操作
│  - 地图中心区域              │     需伸展拇指，可接受
│  - 要素高亮工具栏            │
├──────────────────────────────┤
│  🟢 绿色区（自然区）          │  ← 底部 1/3：高频操作
│  - 底部 Tab 栏               │     拇指自然覆盖范围
│  - 底部 Sheet 面板           │
│  - FAB 悬浮按钮              │
└──────────────────────────────┘
```

**硬性约束**：
- 高频操作（导入、切换 Tab、确认、删除）必须放在绿色区或橙色区底部
- 红色区仅允许放置：标题、Logo、主题切换、设置等低频率操作
- 地图控件在移动端必须位于右下角 FAB（绿色区），不得在左上角

#### 2.3 手势定义

| 手势 | 触发条件 | 地图区域 | 面板区域 | 列表区域 |
|------|----------|----------|----------|----------|
| 单指点击 | touchstart + touchend < 300ms, 位移 < 8dp | 选择要素 | 切换 Tab | 选中项 |
| 单指长按 | touchstart 持续 > 500ms, 位移 < 8dp | 选点并弹出气泡（含触觉反馈） | 无操作 | 进入编辑模式 |
| 单指拖拽 | 位移 > 8dp | 平移地图 | 拖拽 Sheet 高度 | 无操作（由滚动接管） |
| 单指滑动 | 快速滑动（速度 > 0.5px/ms） | 惯性平移 | 切换半屏/全屏 | 惯性滚动 |
| 双指捏合 | 两指距离变化 | 缩放地图 | 无操作 | 无操作 |
| 双指旋转 | 两指角度变化 | 旋转地图 | 无操作 | 无操作 |
| 双击 | 两次点击间隔 < 300ms | 放大 1.5x | 无操作 | 编辑项 |
| 左滑 | 单指水平滑动 > 48dp | 无操作 | 无操作 | 显示操作按钮 |

**硬性约束**：
- 地图区域与面板区域的手势必须严格隔离：地图区域只响应地图手势，面板区域只响应面板手势
- 隔离实现方式：面板区域使用 `@touchstart.prevent.stop` 阻止事件冒泡到地图
- 长按必须伴随触觉反馈：`navigator.vibrate(10)`（Android）
- 拖拽阈值（slop）统一为 8dp，防止轻微抖动触发手势

### 3. 动画与过渡量化规范

**依据**：Material Design 3 Motion Guidelines

#### 3.1 时长

| 动画类型 | 持续时间 | 典型场景 |
|----------|----------|----------|
| 微交互 | 100-150ms | 按钮 hover/active 态、图标状态切换、涟漪效果 |
| 小过渡 | 200-250ms | 开关切换、选中态切换、Tooltip 出现/消失 |
| 中等过渡 | 300-400ms | 面板展开/折叠、Sheet 高度切换、弹窗出现/消失 |
| 大过渡 | 400-500ms | 页面切换、全屏模式切换、横竖屏旋转 |
| 地图动画 | 250-350ms | 飞至要素（flyTo）、缩放至范围（fitToView） |

#### 3.2 缓动函数

| 缓动类型 | cubic-bezier | 适用场景 |
|----------|-------------|----------|
| Standard（标准） | `(0.4, 0.0, 0.2, 1)` | 大多数 UI 过渡：面板展开、Tab 切换 |
| Decelerate（减速） | `(0.0, 0.0, 0.2, 1)` | 元素进入屏幕：弹窗打开、Sheet 上滑 |
| Accelerate（加速） | `(0.4, 0.0, 1, 1)` | 元素离开屏幕：弹窗关闭、Sheet 下滑 |
| Sharp（锐利） | `(0.4, 0.0, 0.6, 1)` | 地图飞行动画、缩放动画 |
| Bouncy（弹跳） | `(0.175, 0.885, 0.32, 1.275)` | 仅用于 FAB 展开/收起（强调操作） |

**硬性约束**：
- 所有动画必须使用 `transform` 和 `opacity`，严禁使用 `height` / `width` / `top` / `left` 动画（触发 Layout/Re-layout，导致帧率下降）
- 地图动画使用 OpenLayers 内置的 `view.animate()`，不自定义 CSS 动画
- 尊重用户 `prefers-reduced-motion`：当系统设置为减少动效时，所有动画时长压缩为 0-50ms

## 代码输出规范

### 必须遵守

1. 所有新组件使用 `<script setup lang="ts">` 语法
2. TypeScript 严格模式，禁止 `any`，所有 props/emits 显式类型标注
3. 样式使用 `scoped` SCSS，移动端样式写在 `@media (max-width: 767px)` 内
4. 触控交互使用 `@touchstart` / `@touchmove` / `@touchend`，与 `@mousedown` / `@mousemove` / `@mouseup` 成对出现
5. 安全区域使用 `env(safe-area-inset-*)`，并在不支持的浏览器上 fallback 到 `0px`
6. 所有动画使用 `transform` + `opacity`，禁止 `height` / `width` 动画
7. 所有尺寸值使用设计 token 而非硬编码：`var(--gis-space-md)` 而非 `16px`
8. 地图相关代码必须处理 `dispose()` 清理，防止内存泄漏

### 禁止事项

- 禁止在组件内硬编码 `768px` 断点，统一使用 `useBreakpoint()` 的 `isMobile` computed
- 禁止在移动端使用 `:hover` 效果作为主要交互（可用 `:active` 替代）
- 禁止使用 `position: fixed` 的弹窗在移动端不处理键盘弹出（iOS 键盘会压缩视口）
- 禁止在移动端地图上叠加超过 3 个悬浮按钮（违反 Tufte 数据墨水比）
- 禁止使用 `setTimeout(fn, 0)` 做动画，统一使用 `requestAnimationFrame`
- 禁止在组件内定义新的颜色值，统一使用 CSS 变量
- 禁止在 `v-for` 中使用 `v-if`（违反 Vue 性能最佳实践）

## 工作方式

1. **先诊后治**：每次接到任务，先分析现有代码和相关组件，再给出方案；分析时必须标注违反的理论
2. **渐进改造**：每次只改一个模块（一个组件文件 + 最多一个 composable），确保不引入回归
3. **移动端优先验证**：改造后优先在 375px 视口（iPhone SE 基准）验证，再验证 768px、1024px、1440px
4. **量化验收**：每个输出必须附带可测量的验收标准，不接受"感觉好多了"等主观判断
5. **代码即文档**：关键交互逻辑用注释说明，注释格式：`// 【理论依据】Fitts's Law: 热区 48dp 确保触控效率`

## 项目关键约定（来自 project_memory）

- 每个数据集维护独立地图实例，通过 `v-show` 切换而非销毁重建
- 坐标转换自动创建新数据集并记录转换路径
- 转换路径图使用垂直树形布局，SVG 贝塞尔曲线连接
- `map.dispose()` 必须带 null 检查
- 要素编辑器高度使用 flexbox 而非固定像素
- CRS 变更需重置 workingFeatures

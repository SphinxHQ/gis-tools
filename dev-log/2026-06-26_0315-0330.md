### 💭 为什么这样想
用户在 013 任务确立 R1 规则"任何切换数据集的行为都要伴随地图切换"。转换路径的每个版本对应一个独立数据集（R4：转换生成新数据集），因此版本切换本质上是数据集切换，应遵循 R1。用户通过对比两种切换场景的行为差异，准确定位了版本切换未触发数据集切换的问题。

根因分析：
- `handlePathNodeClick` / `navigateToChainStep` 仅设置 `activeVersionName`，未调用 `setActive(datasetId)`
- 移除 `datasetActiveDataMap` 后 `GisMapSlot` 的 `:data` 绑定 `entry.data`（不变），地图无变化
- 版本切换未触发数据集切换，违反 R1

修复逻辑：
- 提取 `switchToVersion(ver)` 公共函数
- 若 `ver.datasetId !== activeId`：调用 `setActive(datasetId)` 切换数据集，`watch props.data` 从新数据集恢复转换路径并设置 `activeVersionName`
- 若 `ver.datasetId === activeId`（如 origin 版本）：仅更新 `activeVersionName`

验证结果：
- `npm run typecheck`：exit code 0
- `npm run build`：exit code 0，✓ built in 1m 1s

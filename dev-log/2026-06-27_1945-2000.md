# 19:45 - 20:00 | 增加 TopoJSON 类型支持：导入/文本输入/导出全链路打通

### 🧭 用户在想什么
用户希望扩展项目的数据格式支持范围，新增 TopoJSON 类型。核心约束是 TopoJSON 仅作为数据存储类型（导入/导出/文本输入时识别），使用过程中仍统一使用 GeoJSON——这与项目既有的「所有格式 read 时归一化为 GeoJSON.Feature[]」架构完全契合。用户要求编写对应的转换器和判断函数。

### 📦 用户提了什么
- "导入的文件，文本输入，导出的类型要增加 TopoJSON 类型"
- "导入导出要编写对应的转换器判断函数等"
- "类型，但是这个也是仅仅作为数据存储类型。使用过程中还是使用Geojson"

### 💭 为什么这样想
延续项目既有的 DataFormat 抽象架构（每个格式一个 *DataFormat.ts 实现 read/write），新增格式只需在固定位置扩展：GisDataType 枚举、getDataType 判定、createFormatter 分发、具体转换器实现。TopoJSON 作为 D3 生态的拓扑压缩格式，与 GeoJSON 互转有成熟库支持（topojson-client/topojson-server），且其「共享弧段」特性在存储时能显著减小冗余数据体积，符合「存储类型」的定位。

# 22:15 - 22:30 | 绘制可见性修复与显示字段共享

### 🧭 用户在想什么
用户在验证新增要素时发现绘制完成后看不到自己画的东西，怀疑坐标系再次出错。经排查坐标系已正确，实际是 keep=false 导致绘制图形被立即清除。用户进一步要求：结构树指定的显示字段应在要素编辑器中同步生效，保持两处要素名称一致。

### 📦 用户提了什么
- "我绘制完成后。显示已绘制。然后没有看到我绘制的东西。你看看是不是我绘制后取的坐标。你又搞错坐标系了？"
- "指定显示字段之后要素编辑那边也要换成这个"

### 💭 为什么这样想
绘制后看不到图形会让用户产生"坐标错位"的错觉，这是 GIS 交互中最直观的反馈缺失。用户要求显示字段同步到要素编辑器，体现了对"同一数据多视图一致性"的期望——结构树和要素编辑器都是浏览要素的入口，名称显示应统一，避免在两处看到不同的要素名称造成混淆。

### 🛠 做了什么
| 文件路径 | 操作 | 说明 |
|---|---|---|
| src/components/data/GisFeatureTree.vue | 修改 | handleDrawForAdd 的 keep 从 false 改为 true，让绘制图形保留在 display 图层可见；每次绘制前先 emit GisMapCleanDrawEvent 清除上一次残留避免累积。提交时 GisMapCleanDrawEvent 清除 display 图层，新要素通过 watch 链自动迁移到 vector-input 显示 |
| src/components/data/GisFeatureEditor.vue | 修改 | 增加 displayField prop（与结构树共享）；featureList computed 使用 displayField 生成 label，优先级：指定字段值 > name > NAME > 默认"要素"，统一带 #下标后缀，与结构树 inspectFeature 逻辑一致 |
| src/components/data/GisFeatureTree.vue | 修改 | gis-feature-editor 标签传 :display-field="displayField"，将结构树的字段选择同步到要素编辑器 |

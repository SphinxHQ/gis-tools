# 21:30 - 21:45 | 结构树改进与新增要素功能

### 🧭 用户在想什么
用户关注结构树组件的交互细节与功能完整性。一方面注意到滚动条遮挡按钮的视觉问题，另一方面希望强化结构树作为数据浏览入口的能力——通过字段选择动态控制节点显示名称，并补齐"新增要素"这一缺失的编辑能力，使结构树组件从纯浏览升级为可编辑入口。

### 📦 用户提了什么
- "下方的节点 右侧的按钮需要有右边距，因为出现滚动条后。滚动条遮挡了按钮。"
- "树形组件区域上方需要有一个选项区域。用来设置节点显示。也就是选择属性的某个字段 来显示节点的显示名称"
- "节点默认显示名称后面应该有井号加下标"
- "结构树/要素编辑/新增要素 - 增加新增要素。选中后 下方区域变为新增要素面板，里面 操作按钮 有 绘制：点/线/面/清除。下方是 属性填写。属性默认结构和原数据集属性结构一致"

### 💭 为什么这样想
结构树此前仅承担"浏览+JSON编辑"职责，节点名称固定为"要素 #下标"，无法根据业务字段（如名称、编号）快速识别要素。用户希望通过字段选择让节点显示更具业务语义。滚动条遮挡按钮是典型的布局细节问题，影响操作可达性。新增要素功能是要素编辑能力的自然延伸——已有"要素编辑"tab 做存量修改，缺少"新增"入口做增量录入，绘制+属性填写的交互模式与编辑器保持一致。

| 文件路径 | 操作 | 说明 |

| src/components/data/GisFeatureTree.vue | 修改 | 四项改进：(1) .tree-view 右侧 padding 增至 12px、.node-btn 增加 margin-right 应对滚动条遮挡；(2) tree-view 上方新增 .tree-options 区域，el-select 列出 availableFields（从所有 features 提取属性 key 并集）作为节点显示字段；(3) inspectFeature 接受 field 参数，默认显示"要素 #下标"，选字段后显示"字段值 #下标"，模板用 splitLabel 分离为 .key + .node-index 独立渲染；(4) segmented 增加"新增要素"选项，add-view 含绘制点/线/面/清除按钮 + 属性表单，属性结构基于原数据集属性 key 并集按类型推断默认值，提交后写入数据集并更新 store。新增 useGisDataStore、GisMapDrawEvent/GisMapCleanDrawEvent 依赖，treeHeight 偏移量从 40 调整为 88 以容纳 tree-options 区域 |
